# TS Function Types

---

## Concept

In TypeScript, **function types define**:

* What type of parameters a function accepts
* What type of value it returns

This ensures:

* Correct function usage
* Compile-time error checking
* Predictable behaviour

In short:

> Function types = Contract of a function

---

## Basic Function Syntax in TypeScript

```ts
function functionName(param1: type, param2: type): returnType {
  return value;
}
```

---

# 1. Named Functions

## Concept

A **named function** is declared using the `function` keyword and has a fixed name.

### Example

```ts
function add(a: number, b: number): number {
  return a + b;
}

console.log(add(3, 4));
```

### Explanation

* `a`, `b` must be numbers
* Return type is explicitly `number`
* Passing wrong types → compile-time error

### Output

```
7
```

---

# 2. Anonymous Functions

## Concept

Anonymous functions:

* Do not have a name
* Are assigned to variables
* Useful for short-lived logic

### Example

```ts
const subtract = function (a: number, b: number): number {
  return a - b;
};

console.log(subtract(5, 2));
```

### Explanation

* Function has no name
* Variable `subtract` holds the function
* Type safety still applies

### Output

```
3
```

---

# 3. Arrow Functions

## Concept

Arrow functions are:

* Shorter syntax
* Commonly used in callbacks
* Lexically bind `this`

### Syntax

```ts
const fn = (params): returnType => expression;
```

### Example

```ts
const multiply = (a: number, b: number): number => a * b;

console.log(multiply(2, 5));
```

### Output

```
10
```

### Why Arrow Functions Are Preferred

* Cleaner syntax
* Better readability
* Safe `this` handling

# 4. Optional & Default Parameters

---

## Optional Parameters (`?`)

### Concept

Optional parameters may or may not be passed.

### Rule

Optional parameters must come **after required ones**.

### Example

```ts
function greet(firstName: string, lastName?: string): string {
  return lastName
    ? `Hello ${firstName} ${lastName}`
    : `Hello ${firstName}`;
}

console.log(greet("John"));
console.log(greet("John", "Doe"));
```

---

## Default Parameters

### Concept

Default parameters provide a fallback value.

### Example

```ts
function greet(firstName: string, lastName: string = "Doe"): string {
  return `Hello ${firstName} ${lastName}`;
}

console.log(greet("John"));
console.log(greet("Joe", "Smith"));
```

### Output

```
Hello John Doe
Hello Joe Smith
```

# 5. Return Types

## Concept

Return type specifies **what a function must return**.

### Example

```ts
function square(num: number): number {
  return num * num;
}

console.log(square(4));
```

### Output

```
16
```

### Why Explicit Return Types Matter

* Prevent accidental wrong returns
* Improve readability
* Help refactoring

---

# 6. Void Return Type

## Concept

`void` means:

> This function returns nothing.

### Example

```ts
function logMessage(message: string): void {
  console.log(message);
}

logMessage("Hello, Rahul!");
```

### Output

```
Hello, Rahul!
```

Used for:

* Logging
* Side effects
* DOM updates

---

# 7. Rest Parameters

## Concept

Rest parameters allow variable number of arguments.

### Syntax

```ts
function fn(...args: type[]): returnType {}
```

### Example

```ts
function sum(...numbers: number[]): number {
  return numbers.reduce((acc, curr) => acc + curr, 0);
}

console.log(sum(1, 2, 3, 4, 5));
```

### Output

```
15
```

# 8. Function Overloading

## Concept

Function overloading allows:

* Multiple function signatures
* Single implementation

### Syntax Pattern

```ts
function fn(a: type): type;
function fn(a: type, b: type): type;
function fn(a: any, b?: any): any {
  // implementation
}
```

### Example

```ts
function greet(person: string): string;
function greet(person: string, age: number): string;

function greet(person: string, age?: number): string {
  if (age !== undefined) {
    return `Hello ${person}, you are ${age} years old!`;
  }
  return `Hello ${person}!`;
}

console.log(greet("Anne"));
console.log(greet("John", 30));
```

### Output

```
Hello Anne!
Hello John, you are 30 years old!
```

# 9. Callback Functions

## Concept

A **callback** is:

> A function passed as an argument to another function.

Very common in:

* Async code
* Events
* APIs

---

### Example

```ts
function performOperation(
  a: number,
  b: number,
  callback: (result: number) => void
): void {
  const result = a + b;
  callback(result);
}

performOperation(3, 4, (result) => {
  console.log(result);
});
```

### Output

```
7
```

### Explanation

* Callback enforces parameter & return type
* Prevents misuse
