# PROBLEM (Original Statement)

Given an integer array `nums` and an integer `val`, remove all occurrences of `val` in-place and return the number of elements that are not equal to `val`.

The relative order of the elements may be changed.

It doesn't matter what you leave beyond the returned length.

---

## Topics / Patterns

**Topic:** Arrays
Because we are modifying elements inside an array in-place.

**Patterns Involved:**

1. Two Pointer Pattern
   - Used to overwrite unwanted values.
   - Helps solve in O(n) without extra space.

2. In-place Modification
   - Constraint forces O(1) extra space.

---

## BREAKDOWN PROBLEM

In simple words:

We are given:

- An array
- A value `val`

We must:

- Remove all elements equal to `val`
- Do it in-place
- Return new length

Important:

- We do NOT need to shrink array physically.
- Only first `k` elements must be valid.
- Order is not required to be preserved.

---

## CONSTRAINT UNDERSTANDING

Typical constraints:

- 0 ≤ nums.length ≤ 10⁴
- -10⁴ ≤ nums[i] ≤ 10⁴
- -10⁴ ≤ val ≤ 10⁴

Important implications:

- O(n²) may pass for small n but not ideal.
- In-place required → No extra array.
- Best solution should be O(n) time, O(1) space.

---

# BRUTE FORCE APPROACH

## First Thought Intuition

First natural thought:

"Let me create a new array and copy all elements except `val`."

Why natural?
Because filtering is intuitive.

---

## Thought Process (Step-wise)

1. Create empty array `result`
2. Traverse nums
3. If nums[i] != val → push into result
4. Copy result back into nums
5. Return result.size()

---

## Pseudocode

```
create empty result
for each element in nums:
    if element != val:
        add element to result

copy result back to nums
return size of result
```

---

## Algorithm

1. Initialize empty vector.
2. Loop through array.
3. Store valid elements.
4. Copy back.
5. Return size.

---

## CODE

### C++

#### V1 (Only Required Function)

```cpp
int removeElement(vector<int>& nums, int val) {
    vector<int> temp;  // Extra space used

    // Traverse entire array
    for (int i = 0; i < nums.size(); i++) {
        // If current element is NOT equal to val
        if (nums[i] != val) {
            temp.push_back(nums[i]);  // Keep it
        }
    }

    // Copy back to original array
    for (int i = 0; i < temp.size(); i++) {
        nums[i] = temp[i];
    }

    // Return new length
    return temp.size();
}
```

---

#### V2 (Complete Program)

```cpp
#include <iostream>
#include <vector>
using namespace std;

int removeElement(vector<int>& nums, int val) {
    vector<int> temp;  // Stores valid elements

    // Step 1: Traverse array
    for (int i = 0; i < nums.size(); i++) {

        // If current element is not equal to val
        if (nums[i] != val) {
            temp.push_back(nums[i]);  // Keep element
        }
    }

    // Step 2: Copy valid elements back
    for (int i = 0; i < temp.size(); i++) {
        nums[i] = temp[i];
    }

    // Step 3: Return new length
    return temp.size();
}

int main() {
    vector<int> nums = {3, 2, 2, 3};
    int val = 3;

    int newLength = removeElement(nums, val);

    cout << "New Length: " << newLength << endl;

    cout << "Modified Array: ";
    for (int i = 0; i < newLength; i++) {
        cout << nums[i] << " ";
    }

    return 0;
}
```

---

## Time and Space Complexity

Time Complexity:
O(n) → one traversal + copy

Space Complexity:
O(n) → extra temp array

---

## Dry Run

Input:
nums = [3,2,2,3]
val = 3

temp after loop:
[2,2]

Copy back:
nums = [2,2,?,?]

Return 2

---

## Edge Cases

1. Empty array → return 0
2. All elements equal to val → return 0
3. No element equal to val → return n
4. Single element array

Handled correctly.

---

## How This Approach Handles the Problem

It filters unwanted values and overwrites original array.

---

## Does This Approach Fail?

Yes.

It violates constraint:

- Requires O(1) extra space
- Uses O(n) extra space

So we move to better approach.

# BETTER APPROACH

## First Thought Intuition

Instead of extra array, why not overwrite inside same array?

Idea:
Maintain a pointer `k` where next valid element should go.

---

## Thought Process (Step-wise)

1. Initialize k = 0
2. Traverse array
3. If nums[i] != val:
   - nums[k] = nums[i]
   - k++

4. Return k

This shifts valid elements forward.

---

## Pseudocode

```
k = 0
for i from 0 to n-1:
    if nums[i] != val:
        nums[k] = nums[i]
        k++

return k
```

---

## Algorithm

Single pass:

- Keep writing non-val elements at front.

---

## CODE

### C++

#### V1

```cpp
int removeElement(vector<int>& nums, int val) {
    int k = 0;  // Position to place next valid element

    for (int i = 0; i < nums.size(); i++) {

        // If current element is NOT equal to val
        if (nums[i] != val) {

            // Place element at index k
            nums[k] = nums[i];

            // Move k forward
            k++;
        }
    }

    // k is count of valid elements
    return k;
}
```

---

#### V2

```cpp
#include <iostream>
#include <vector>
using namespace std;

int removeElement(vector<int>& nums, int val) {

    int k = 0;  // Pointer for valid position

    // Traverse entire array
    for (int i = 0; i < nums.size(); i++) {

        // If element is NOT equal to val
        if (nums[i] != val) {

            nums[k] = nums[i];  // Move element forward
            k++;                // Increment valid counter
        }
    }

    return k;  // Number of elements not equal to val
}

int main() {

    vector<int> nums = {3, 2, 2, 3};
    int val = 3;

    int newLength = removeElement(nums, val);

    cout << "New Length: " << newLength << endl;

    cout << "Modified Array: ";
    for (int i = 0; i < newLength; i++) {
        cout << nums[i] << " ";
    }

    return 0;
}
```

---

## Time and Space Complexity

Time: O(n)
Space: O(1)

---

## Dry Run

nums = [3,2,2,3]
val = 3

i=0 → skip
i=1 → nums[0]=2
i=2 → nums[1]=2
i=3 → skip

Final:
[2,2,?,?]
k=2

---

## Edge Cases

All covered correctly.

---

## How This Approach Handles the Problem

It reuses same array.
No extra space.
Single traversal.

---

## Does This Approach Fail?

No.

It satisfies constraints.

Since order does not matter and O(1) space achieved, this is already optimal.

---

# OPTIMAL APPROACH

For this problem, Better Approach is already Optimal.

No further optimization possible.

---

## Why This Is Optimal

Constraints demand:

- In-place
- O(1) space
- O(n) time

We:

- Traverse once → O(n)
- No extra memory → O(1)

Cannot do better than O(n) because we must inspect each element.

# Pattern Recognition Mindset

When you see:

- "Remove element"
- "In-place"
- "Return new length"
- "Order doesn't matter"

Immediately think:

→ Two Pointer Overwrite Pattern

This pattern also applies to:

- Move Zeros
- Remove Duplicates from Sorted Array
- Partition array by condition
- Stable filtering problems
