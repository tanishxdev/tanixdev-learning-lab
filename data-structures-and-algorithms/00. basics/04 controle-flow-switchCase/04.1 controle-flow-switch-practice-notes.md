# SOLUTIONS (WITH THINKING PROCESS)

## Q1 Solution: Day of Week

### Thinking

* One variable (`day`)
* Fixed values (1â€“7)
* Perfect case for `switch`

### Code

```cpp
int day;
cin >> day;

switch(day)
{
    case 1: cout << "Monday"; break;
    case 2: cout << "Tuesday"; break;
    case 3: cout << "Wednesday"; break;
    case 4: cout << "Thursday"; break;
    case 5: cout << "Friday"; break;
    case 6: cout << "Saturday"; break;
    case 7: cout << "Sunday"; break;
    default: cout << "Invalid day";
}
```

## Q2 Solution: Simple Calculator

### Thinking

* Operator is a single character
* Switch works well with `char`
* Handle division carefully

### Code

```cpp
int a, b;
char op;
cin >> a >> op >> b;

switch(op)
{
    case '+': cout << a + b; break;
    case '-': cout << a - b; break;
    case '*': cout << a * b; break;
    case '/':
        if(b != 0)
            cout << a / b;
        else
            cout << "Division by zero";
        break;
    default:
        cout << "Invalid operator";
}
```

## Q6 Solution: Missing `break` Demo

### Thinking

* Without `break`, execution falls through
* Used intentionally sometimes, but risky

### Code

```cpp
int x = 2;

switch(x)
{
    case 1: cout << "One";
    case 2: cout << "Two";
    case 3: cout << "Three";
}
```

### Output

```
TwoThree
```

### Why?

* Case 2 matched
* No `break`
* Case 3 also executed

## FINAL REMEMBER POINTS (INTERVIEW)

* `switch` works with **fixed constant values**
* Always use `break` unless fall-through is intended
* `default` is mandatory for safety
* Use `if-else` for **ranges**, `switch` for **menus**
