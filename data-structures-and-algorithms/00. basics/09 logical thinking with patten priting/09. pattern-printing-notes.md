# PATTERNS PRINTING

Patterns are not just printing stars.
They train you to:

* Control loops
* Understand row-column relationships
* Build algorithmic thinking

Rule:
ðŸ‘‰ Every pattern depends on:
Row number (i)
Column number (j)

---

## HOW TO THINK FOR ANY PATTERN

Always ask:

1. How many rows?
2. What to print in each row?
3. How many columns per row?
4. How does it change each line?

General skeleton:

```cpp
for(int i=1; i<=n; i++) {        // rows
    for(int j=1; j<=something; j++) {   // columns
        print;
    }
    cout << endl;
}
```

# HOW TO MASTER PATTERNS (Rule)

Every star pattern boils down to:

1. Spaces logic
2. Character logic
3. Relationship with row index

---

# MASTER TEMPLATE CODE

```cpp
int n = 5;

for(int i = 1; i <= n; i++){
    for(int j = 1; j <= n; j++){
        cout << "*";
    }
    cout << endl;
}
```

You modify inner loop logic based on pattern need.
