# MULTIPLE COMPONENTS

(Problem → Concept → Folder → Code → Explanation → Output)

---

## 1. Problem Without Multiple Components

If everything is placed inside a single file:

* One huge `App.jsx`
* Difficult to understand logic
* No separation of responsibilities
* Reusability becomes impossible
* Maintenance becomes risky

Real problem:
One developer changes header → entire app can break.

---

## 2. Core Concept

Multiple components mean:

> Break UI into small, independent pieces and combine them together.

Each component handles:

* One clear responsibility
* One logical UI part

This follows:
Separation of Concerns Principle.

---

## 3. Real World Intuition

Website is not one block.

It contains:

* Header
* Sidebar
* Content
* Footer

React allows each of these to be separate components.

---

## 4. Folder Structure

```
src/
  components/
    Header.jsx
    Profile.jsx
    Footer.jsx
  App.jsx
  main.jsx
```

---

## 5. Code Implementation

### A. Header Component

**src/components/Header.jsx**

```jsx
// Header.jsx
// Responsible only for top heading section.

function Header() {
  return (
    <header>
      <h2>User Dashboard</h2>
    </header>
  );
}

export default Header;
```

---

### B. Profile Component

**src/components/Profile.jsx**

```jsx
// Profile.jsx
// Displays user profile data.

function Profile() {
  return (
    <section>
      <p>Name: Tanish</p>
      <p>Role: Frontend Developer</p>
    </section>
  );
}

export default Profile;
```

---

### C. Footer Component

**src/components/Footer.jsx**

```jsx
// Footer.jsx
// Bottom section of the page.

function Footer() {
  return (
    <footer>
      <p>© 2025 React Learning</p>
    </footer>
  );
}

export default Footer;
```

---

### D. Combining in App.jsx

**src/App.jsx**

```jsx
import Header from "./components/Header";
import Profile from "./components/Profile";
import Footer from "./components/Footer";

function App() {
  return (
    <div>
      <Header />
      <Profile />
      <Footer />
    </div>
  );
}

export default App;
```

---

## 6. How It Works

1. App.jsx imports 3 components
2. Each component renders independently
3. React merges them into a single UI
4. UI becomes modular and clean

Execution Flow:

```
App → Header  
    → Profile  
    → Footer
```

## 8. Why Multiple Components Are Important

| Benefit         | Result                               |
| --------------- | ------------------------------------ |
| Modularity      | Split complex UI                     |
| Reusability     | Use components anywhere              |
| Maintainability | Easy bug fixing                      |
| Team Work       | Multiple devs can work independently |
| Scalability     | Suitable for large apps              |

---

## 9. Comparison

One File Approach vs Multi Components

Single File:

* Hard to debug
* Huge code
* Confusing logic

Multiple Components:

* Clean structure
* Predictable flow
* Easier testing
