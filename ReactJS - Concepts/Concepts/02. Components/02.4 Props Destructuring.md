## 1. Problem Without Destructuring

Using props like this:

```jsx
function UserCard(props) {
  return (
    <div>
      <p>{props.name}</p>
      <p>{props.age}</p>
      <p>{props.role}</p>
    </div>
  );
}
```

Problems:

* Repetitive `props.` everywhere
* Less readable
* Messy for large components
* Hard to scale

---

## 2. Core Concept

Prop Destructuring means:

> Extracting properties from props object into variables.

This makes code:

* Cleaner
* Shorter
* More readable
* Professional standard

---

## 3. Real-World Intuition

Instead of:

```
person.name
person.age
person.role
```

You directly write:

```
name
age
role
```

Same data, cleaner access.

## 5. Code Implementation

### A. Child Component with Prop Destructuring

**src/components/UserCard.jsx**

```jsx
// UserCard.jsx
// Demonstrates prop destructuring.

function UserCard({ name, age, role }) {
  return (
    <div>
      <h3>User Information</h3>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
      <p>Role: {role}</p>
    </div>
  );
}

export default UserCard;
```

Explanation:

* Instead of props, we directly extract:

  * name
  * age
  * role
* Cleaner usage inside JSX.

---

### B. Parent Component (Same as before)

**src/App.jsx**

```jsx
import UserCard from "./components/UserCard";

function App() {
  return (
    <div>

      <UserCard

      <UserCard 
        name="Rahul"
        age={23}
        role="Backend Developer"
      />

}

export default App;
```

---

## 6. How It Works Internally

React passes:

```js
props = { name: "Tanish", age: 21, role: "Frontend Developer" }
```

Destructuring converts it to:

```js
const name = props.name;
const age = props.age;
const role = props.role;
```

But automatically in function parameters.
