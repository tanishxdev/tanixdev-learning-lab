## 6. Simple Example

## 1. Why was React created?

React was created to solve the problem of **complex and hard-to-manage UI updates** in large web applications.

Before React:

* Developers manually updated the DOM.
* Updating one small part of UI required rewriting large logic.
* Code became messy and hard to maintain.

React introduced a better way:

* UI automatically updates when data changes.
* Developer only focuses on ‚Äúwhat to show‚Äù, not ‚Äúhow to update‚Äù.

In short:
React was created to make building **dynamic, fast, and scalable UI easier**.

## 2. What problem does React solve in web development?

React solves:

* Inefficient DOM manipulation
* Spaghetti UI logic
* Hard-to-sync data and UI
* Performance issues in large apps

It provides:

* Automatic UI updates
* Structured component system
* Predictable data flow
* Faster rendering

So the core problem solved:
Keeping UI and data in sync in a clean and efficient way.

## 3. Difference between DOM and Virtual DOM?

| DOM                         | Virtual DOM               |
| --------------------------- | ------------------------- |
| Real browser structure      | Lightweight JS copy       |
| Slow to update              | Fast to update            |
| Direct browser manipulation | React works on copy first |
| Repaints entire sections    | Updates only changed part |

Flow:

```
State Change ‚Üí Virtual DOM Update ‚Üí Compare ‚Üí Update Real DOM
```

Key idea:
React avoids expensive DOM operations by using Virtual DOM as a middle layer.

## 4. What is declarative programming in React?

Declarative means:
You describe **what UI should look like**, not how to build it step-by-step.

Example:

```jsx
{isLoggedIn && <Dashboard />}
```

You just say:
"If logged in, show Dashboard"

React decides:

* When to render
* How to update
* Which parts to change

This makes code:

* Cleaner
* Easier to understand
* Less error-prone

## 5. How does React improve performance?

React improves performance by:

* Using Virtual DOM
* Updating only changed elements
* Efficient diffing algorithm
* Component reuse
* Lazy loading and code splitting

Instead of reloading full UI:
It updates only the modified elements.

This reduces:

* Browser reflow
* Repaint cost
* Unnecessary computations

State-driven UI means:
UI depends entirely on the state.

Change state ‚Üí UI updates automatically.

React automatically re-renders the affected UI.

So:
UI is not manually controlled.
It reacts to state changes.

React focuses only on:

It does NOT provide:

## 10. What happens when state changes in React?

Flow when state changes:

1. State updates via setState / useState
2. React re-runs component function
3. New Virtual DOM created
4. Old and new Virtual DOM compared (diffing)
5. Only changed elements updated in Real DOM
6. UI visibly updates

This is why React is fast and efficient.

# 3. Difference between DOM and Virtual DOM?

## Concepts

| Feature   | DOM                    | Virtual DOM            |
| --------- | ---------------------- | ---------------------- |
| Type      | Browser tree structure | In-memory copy         |
| Speed     | Slow to update         | Fast updates           |
| Update    | Direct manipulation    | Diff + minimal changes |
| Rendering | Full reflow            | Partial re-render      |

React always updates the Virtual DOM first, then efficiently updates only changed parts in the real DOM.

## Interview Answer

DOM is the real browser UI tree, while Virtual DOM is an in-memory lightweight copy. React updates the Virtual DOM first, performs diffing, and updates only the changed elements in the real DOM for better performance.

You declare **UI = f(state)**

Imperative approach (plain JS):

React simplifies this.

## Interview Answer

Declarative programming means describing the UI based on the current state instead of manually manipulating the DOM. React updates the DOM for you based on state changes.
