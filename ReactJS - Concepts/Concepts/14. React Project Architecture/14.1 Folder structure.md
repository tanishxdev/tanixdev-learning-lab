# 14.1 WHY PROJECT ARCHITECTURE MATTERS

Beginner React apps work with simple structure:

```
src/
  App.jsx
  components/
```

But real-world apps quickly become large and complex.

Example:

* Authentication
* Product listing
* Cart
* User profile
* Admin panel
* Dashboards
* Filters
* Pagination
* Services
* Hooks
* Forms
* Routers
* API integrations

A messy structure leads to problems:

1. Hard to find code
2. Hard to debug
3. Hard to scale features
4. Hard to maintain
5. Hard for team collaboration
6. High chance of bugs
7. Re-renders and performance issues

Clean architecture solves all these.

---

# 14.2 THE GOAL OF ARCHITECTURE

A good architecture should:

* Separate concerns
* Group related files
* Make features independent
* Make testing easier
* Improve readability
* Support scaling to 50+ components
* Create a predictable folder structure

---

# 14.3 THE MODERN REACT PROJECT STRUCTURE (STANDARD)

Here is the folder structure used by most companies today:

```
src/
├── assets/               # Images, icons, fonts
├── components/           # Reusable UI components
├── pages/                # Route-level pages
├── features/             # Full feature modules
│   ├── auth/
│   ├── cart/
│   ├── products/
│   └── user/
├── hooks/                # Custom hooks
├── context/              # Context providers
├── store/                # Redux / Zustand stores
├── services/             # API calls (axios/fetch)
├── utils/                # Helper functions
├── router/               # React Router setup
├── layouts/              # Layouts (Navbar, Sidebar)
├── styles/               # Global CSS files
└── App.jsx
```

We will build each part.

But first, let’s understand the separation logic.

---

# 14.4 COMPONENTS VS PAGES

## Pages:

* Connected to routing
* Represent full screens
* Example: Home, Products, Cart, Login
* Should NOT contain small UI components
* Should assemble sections

Example:

```
src/pages/HomePage.jsx
src/pages/ProductPage.jsx
```

## Components:

* Small reusable blocks
* Buttons, Cards, Modals, Inputs
* Used inside pages or other components

Example:

```
src/components/Button.jsx
src/components/ProductCard.jsx
src/components/Modal.jsx
```

Pages use components, not the other way around.

This separation makes the project clean.

---

# 14.5 SHARED COMPONENTS DESIGN

Shared components are UI building blocks you use everywhere.

Examples:

* Button
* Input
* Navbar
* Footer
* Modal
* Card
* Loader
* Dropdown
* Tabs

Folder:

```
src/components/
```

Example file:

`src/components/Button.jsx`

```jsx
export default function Button({ children, onClick }) {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
}
```

These components must be:

* Reusable
* Stateless (UI only)
* No business logic

---

# 14.6 FEATURES LAYER (The Most Important Part)

This is the modern industry approach.

Each feature gets its own folder.

Example:

```
src/features/auth/
  LoginForm.jsx
  SignupForm.jsx
  authService.js
  authSlice.js
  useAuth.js

src/features/cart/
  CartPage.jsx
  cartSlice.js
  CartItem.jsx
  cartService.js
```

This isolates features:

* Cart code inside cart folder
* Auth code inside auth folder
* Product code inside products folder

This is called **modular architecture**.

---

# 14.7 HOOKS LAYER (Custom Hooks)

Every reusable logic goes here.

```
src/hooks/useFetch.js
src/hooks/useToggle.js
src/hooks/useLocalStorage.js
src/hooks/useAuth.js
```

Example:

`src/hooks/useToggle.js`

```jsx
import { useState } from "react";

export default function useToggle(initial = false) {
  const [value, setValue] = useState(initial);
  const toggle = () => setValue((v) => !v);
  return [value, toggle];
}
```

This avoids repeating logic across components.

---

# 14.8 CONTEXT LAYER

Global logic that affects many components.

Folder:

```
src/context/UserContext.jsx
src/context/ThemeContext.jsx
src/context/AuthContext.jsx
```

Example:

`src/context/ThemeContext.jsx`

```jsx
import { createContext, useState } from "react";

export const ThemeContext = createContext();

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}
```

Router + ThemeProvider + AuthProvider wrap the entire app.

---

# 14.9 STATE LAYER (REDUX / ZUSTAND)

Advanced global state.

Folder:

```
src/store/
  store.js
  userSlice.js
  cartSlice.js
```

Why separate store?

Keeps global logic in one place.

---

# 14.10 SERVICES LAYER (API CALLS)

All API calls must be in a service file.

```
src/services/authService.js
src/services/productService.js
src/services/userService.js
```

Example:

```jsx
import axios from "axios";

const API = "https://api.example.com";

export const loginUser = async (data) => {
  return axios.post(`${API}/login`, data);
};
```

This keeps API logic out of components.

---

# 14.11 UTILS / HELPERS

Common functions used everywhere.

```
src/utils/formatDate.js
src/utils/currency.js
src/utils/debounce.js
src/utils/validators.js
```

Example:

`src/utils/currency.js`

```jsx
export const formatCurrency = (num) => {
  return new Intl.NumberFormat("en-US", {
    style: "currency",
    currency: "USD",
  }).format(num);
};
```

---

# 14.12 ROUTER LAYER

Dedicated file for routing.

```
src/router/AppRouter.jsx
```

Example:

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "../pages/HomePage";

export default function AppRouter() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
      </Routes>
    </BrowserRouter>
  );
}
```

App.jsx becomes clean:

```jsx
import AppRouter from "./router/AppRouter";

export default function App() {
  return <AppRouter />;
}
```

---

# 14.13 LAYOUTS LAYER

Pages often share:

* Navbar
* Sidebar
* Footer

Instead of copying them, use a layout.

`src/layouts/MainLayout.jsx`

```jsx
import Navbar from "../components/Navbar";

export default function MainLayout({ children }) {
  return (
    <div>
      <Navbar />
      <main>{children}</main>
    </div>
  );
}
```
