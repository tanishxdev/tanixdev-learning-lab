# CONDITIONAL RENDERING

(Problem → Concept → Folder → Code → Explanation → Output)

---

## 1. Problem Without Conditional Rendering

If UI cannot change based on conditions, then:

* Same content shown to everyone
* No login/logout logic
* No error/success messages
* No role-based UI
* No dynamic control

Example problem:
Even if user is not logged in, dashboard is still visible.

---

## 2. Core Concept

Conditional Rendering means:

> Showing different UI based on conditions.

React uses normal JavaScript logic:

* if / else
* ternary operator
* logical &&

JS decides what JSX should be displayed.

---

## 3. Real-World Intuition

If user is logged in → Show Dashboard
If user not logged in → Show Login Button

This decision is conditional.

## 5. Code Implementation

### A. Conditional Rendering Component

**src/components/Conditional.jsx**

```jsx
// Conditional.jsx
// Demonstrates rendering UI based on conditions.

function Conditional() {

  const isLoggedIn = true;
  const userRole = "admin";

  return (
    <div>
      <h2>Conditional Rendering Example</h2>

      {/* Using if-like logic with ternary */}
      {isLoggedIn 
        ? <p>Welcome back, user!</p>
        : <p>Please login</p>
      }

      {/* Logical AND rendering */}
      {userRole === "admin" && (
        <p>Admin Dashboard Access Granted</p>
      )}
    </div>
  );
}

export default Conditional;
```

### B. Using Component

**src/App.jsx**

```jsx
import Conditional from "./components/Conditional";

function App() {
  return (

      <Conditional />
    </div>
  );
}

export default App;
```

## 6. Ways to Do Conditional Rendering

### ✅ Ternary Operator

```jsx
condition ? <ComponentA /> : <ComponentB />
```

### ✅ Logical AND

```jsx
condition && <Component />
```

### ✅ if Statement (outside JSX)

```jsx
let message;
if(isLoggedIn) {
  message = <p>Welcome</p>;
}
```
