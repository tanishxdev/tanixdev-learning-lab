# DYNAMIC CONTENT

(Problem → Concept → Folder → Code → Explanation → Output)

---

## 1. Problem Without Dynamic Content

If UI is static (hardcoded text), then:

* Cannot change data at runtime
* Same content always appears
* Not suitable for real apps
* No user-based personalization
* No data-driven UI

Example problem:

```jsx
<p>Welcome User</p>
```

This will always show **User**, not real names.

---

## 2. Core Concept

Dynamic content means:

> Rendering data that can change at runtime.

In React, this is done using:

* JavaScript expressions inside `{}`
* Variables
* Functions
* State values
* Props values

---

## 3. Real-World Intuition

Static:

Dynamic:

The UI changes based on data.

## 5. Code Implementation

### A. Dynamic Content Component

**src/components/DynamicContent.jsx**

```jsx
// DynamicContent.jsx
// Demonstrates how React renders changing data.

function DynamicContent() {

  // Dynamic data
  const userName = "Tanish";
  const age = 21;
  const isLoggedIn = true;

  return (
    <div>
      <h2>Dynamic Content Example</h2>

      {/* Variable rendering */}
      <p>Name: {userName}</p>
      <p>Age: {age}</p>

      {/* Expression rendering */}
      <p>Next Year Age: {age + 1}</p>

      {/* Boolean condition display */}
      <p>Status: {isLoggedIn ? "Online" : "Offline"}</p>
    </div>
  );
}

export default DynamicContent;
```

---

### B. Using in App.jsx

**src/App.jsx**

```jsx
import DynamicContent from "./components/DynamicContent";

function App() {
  return (
    <div>
      <h1>Dynamic Content Concept</h1>
      <DynamicContent />
    </div>
  );
}

export default App;
```

## 6. How Dynamic Rendering Works

Flow:

```
JS Variable → JSX {} → Virtual DOM → UI
```

When data changes:

* React re-runs component
* UI updates automatically

## 8. What You Can Render Dynamically

Inside `{}` you can render:

* Variables
* Expressions
* Function return values
* Props
* State values

Examples:

```jsx
<p>{10 + 20}</p>
<p>{username.toUpperCase()}</p>
<p>{getStatus()}</p>
```
