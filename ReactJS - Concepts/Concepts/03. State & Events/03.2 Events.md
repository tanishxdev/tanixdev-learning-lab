## 1. Problem Without Events

Without events:

* No interaction
* Buttons do nothing
* Forms cannot submit
* UI is static
* User actions ignored

Modern apps require:

* Click handling
* Input tracking
* Keyboard actions
* Mouse actions

---

## 2. Core Concept

React Events are:

> Functions that run when user interacts with UI.

They are handled using:

* camelCase event names
* Function references (not direct calls)

Examples:

* onClick
* onChange
* onSubmit
* onMouseEnter

---

## 3. Real-World Intuition

User clicks button → Some logic runs
User types input → Data updates
User submits form → Validation happens

Event = Reaction to User Action

## 5. Code Implementation

### A. Event Handling Component

**src/components/EventDemo.jsx**

```jsx
// EventDemo.jsx
// Demonstrates event handling in React.

function EventDemo() {

  // Event handler function
  function handleClick() {
    alert("Button was clicked!");
  }

  // Event with parameter
  function showMessage(name) {
    alert("Hello " + name);
  }

  return (
    <div>
      <h2>React Events Example</h2>

      {/* Basic click event */}
      <button onClick={handleClick}>
        Click Me
      </button>

      {/* Passing parameter */}
      <button onClick={() => showMessage("Tanish")}>
        Greet User
      </button>
    </div>
  );
}

export default EventDemo;
```

## 6. How React Events Work

Flow:

```
User Action → Event Trigger → Handler Function Executes → UI/Logic Updates
```

Important:

* Do NOT write `onClick={handleClick()}`
* This will execute immediately.

Correct:

```
onClick={handleClick}
```

---

## 7. Common React Events

| Event       | Trigger      |
| ----------- | ------------ |
| onClick     | Mouse click  |
| onChange    | Input change |
| onSubmit    | Form submit  |
| onKeyDown   | Key press    |
| onMouseOver | Hover        |

## 8. Event with Event Object

```jsx
function handleInput(e) {
  console.log(e.target.value);
}

<input onChange={handleInput} />
```
