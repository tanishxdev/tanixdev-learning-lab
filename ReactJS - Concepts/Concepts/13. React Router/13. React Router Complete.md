# 13.1 WHAT IS ROUTING? WHY IS IT NEEDED?

## Problem Before Router:

React is a **Single Page Application**.

Meaning:

* Only one HTML file (index.html)
* React controls everything
* If you go to /about or /contact, the browser tries to load a new page
* But React apps don’t have multiple HTML pages

So, without a router:

* URL cannot change meaningfully
* User cannot navigate to multiple pages
* No deep linking (cannot send URL to others)
* No page reload prevention
* No history management

Companies need:

* Login page
* Dashboard
* Product page
* User profile
* Admin pages

All with proper URLs.

---

# Solution: React Router

React Router:

* Listens to URL changes
* Shows the correct component based on URL
* Prevents page reload
* Manages browser history
* Gives navigation functions
* Allows dynamic URLs (/user/123)
* Handles protected routes

It is the most popular routing library in React.

# 13.3 Basic Setup (BrowserRouter + Routes + Route)

Create folder:

`src/components/13.ReactRouter`

Now create three simple pages:

`Home.jsx`

```jsx
export default function Home() {
  return <h2>Home Page</h2>;
}
```

`About.jsx`

```jsx
export default function About() {
  return <h2>About Page</h2>;
}
```

`Contact.jsx`

```jsx
export default function Contact() {
  return <h2>Contact Page</h2>;
}
```

Now connect them in App.jsx:

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./components/13.ReactRouter/Home";
import About from "./components/13.ReactRouter/About";
import Contact from "./components/13.ReactRouter/Contact";

export default function App() {
  return (
    <BrowserRouter>
      <Routes>

        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />

      </Routes>
    </BrowserRouter>
  );
}
```

This is the minimal routing setup.

---

# How It Works Internally

1. BrowserRouter wraps the entire app
2. It listens for URL changes
3. Routes contains all available pages
4. Route maps a URL path to a component

When user goes to:

* `/` → Home
* `/about` → About
* `/contact` → Contact

No reload happens. React swaps components.

# 13.4 Navigation (Link, NavLink, useNavigate)

## A) Link (for page-to-page navigation)

`Navbar.jsx`

```jsx
import { Link } from "react-router-dom";

export default function Navbar() {
  return (
    <nav>
      <Link to="/">Home</Link> | 
      <Link to="/about">About</Link> | 
      <Link to="/contact">Contact</Link>
    </nav>
  );
}
```

Add in App.jsx above Routes:

```jsx
<Navbar />
```

Link replaces the anchor tag (`<a>`) to prevent page reload.

---

## B) NavLink (active style)

NavLink adds an “active” class automatically.

```jsx
<NavLink
  to="/about"
  style={({ isActive }) => ({
    fontWeight: isActive ? "bold" : "normal"
  })}
>
  About
</NavLink>
```

Common for sidebars and navigation menus.

---

## C) useNavigate (go programmatically)

Example: Redirect after login.

`Login.jsx`

```jsx
import { useNavigate } from "react-router-dom";
import { useState } from "react";

export default function Login() {
  const [user, setUser] = useState("");
  const navigate = useNavigate();

  const handleLogin = () => {
    if (user.trim() !== "") {
      navigate("/dashboard");
    }
  };

  return (
    <div>
      <input onChange={(e) => setUser(e.target.value)} />
      <button onClick={handleLogin}>Login</button>
    </div>
  );
}
```

`navigate("/dashboard")` changes the page without reload.

# 13.5 URL Params (Dynamic Routing)

Used for:

* Product pages
* User profiles
* Blog posts

Example URL:

```
/product/101
```

Create file:

`Product.jsx`

```jsx
import { useParams } from "react-router-dom";

export default function Product() {
  const { id } = useParams();

  return <h2>Product ID: {id}</h2>;
}
```

Add route:

```jsx
<Route path="/product/:id" element={<Product />} />
```

Now:

* `/product/1`
* `/product/55`
* `/product/999`

All show different IDs.

# 13.6 Query Params (Filtering, Searching)

Used for:

* Search filters
* Pagination
* Sorting

Example:

```
/products?category=shoes&sort=price
```

Create file:

`FilterPage.jsx`

```jsx
import { useSearchParams } from "react-router-dom";

export default function FilterPage() {
  const [params] = useSearchParams();

  const category = params.get("category");
  const sort = params.get("sort");

  return (
    <div>
      <h2>Category: {category}</h2>
      <h2>Sort: {sort}</h2>
    </div>
  );
}
```

Add route:

```jsx
<Route path="/products" element={<FilterPage />} />
```

# 13.7 Nested Routes

Used for:

* Dashboards
* Settings pages
* Admin sections

Example structure:

```
/dashboard
/dashboard/profile
/dashboard/billing
```

Create:

`Dashboard.jsx`

```jsx
import { Outlet, Link } from "react-router-dom";

export default function Dashboard() {
  return (
    <div>
      <h2>Dashboard</h2>

      <Link to="profile">Profile</Link> |
      <Link to="billing">Billing</Link>

      <Outlet />
    </div>
  );
}
```

Add nested routes:

```jsx
<Route path="/dashboard" element={<Dashboard />}>
  <Route path="profile" element={<h3>Profile Page</h3>} />
  <Route path="billing" element={<h3>Billing Page</h3>} />
</Route>
```

Outlet renders nested pages.

# 13.8 Protected Routes (Authentication)

Real apps protect pages like:

* Dashboard
* Orders
* Admin Panel

Create file:

`ProtectedRoute.jsx`

```jsx
import { Navigate } from "react-router-dom";

export default function ProtectedRoute({ isAuth, children }) {
  if (!isAuth) return <Navigate to="/login" />;

  return children;
}
```

Use in App.jsx:

```jsx
<Route
  path="/dashboard"
  element={
    <ProtectedRoute isAuth={isLoggedIn}>
      <Dashboard />
    </ProtectedRoute>
  }
/>
```

This is essential for any authenticated app.
