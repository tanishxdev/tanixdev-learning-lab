#### BASIC HOOKS

### 1. `useState`

**What:** Manages local component state
**Why:** To store changing data (counter, input, toggle)
**How:** Returns `[state, setState]`

```js
const [count, setCount] = useState(0);
```

---

### 2. `useEffect`

**What:** Runs side effects
**Why:** Fetch data, subscriptions, DOM sync
**How:** Runs after render based on dependencies

```js
useEffect(() => {
  console.log("mounted");
}, []);
```

---

### 3. `useContext`

**What:** Access context value
**Why:** Avoid prop drilling
**How:** Reads nearest Provider value

```js
const theme = useContext(ThemeContext);
```

---

#### STATE / LOGIC HOOKS

### 4. `useReducer`

**What:** State management via reducer
**Why:** Complex state transitions
**How:** `dispatch → reducer → new state`

```js
const [state, dispatch] = useReducer(reducer, 0);
```

---

### 5. `useRef`

**What:** Mutable reference
**Why:** Access DOM / persist value without re-render
**How:** `.current` holds value

```js
const inputRef = useRef(null);
```

---

#### PERFORMANCE HOOKS

### 6. `useMemo`

**What:** Memoizes computed value
**Why:** Avoid expensive recalculation
**How:** Recomputes only when deps change

```js
const value = useMemo(() => heavyCalc(x), [x]);
```

---

### 7. `useCallback`

**What:** Memoizes function
**Why:** Prevent unnecessary re-renders
**How:** Same function reference if deps unchanged

```js
const handleClick = useCallback(() => {}, []);
```

---

#### DOM / LIFECYCLE HOOKS

### 8. `useLayoutEffect`

**What:** Sync effect after DOM mutation
**Why:** Measure DOM before paint
**How:** Runs before browser paints

```js
useLayoutEffect(() => {}, []);
```

---

### 9. `useImperativeHandle`

**What:** Customize ref exposed to parent
**Why:** Control child methods
**How:** Used with `forwardRef`

```js
useImperativeHandle(ref, () => ({ focus }));
```

---

### 10. `useDebugValue`

**What:** Label custom hooks in DevTools
**Why:** Debugging
**How:** Dev-only metadata

```js
useDebugValue(value);
```

---

#### REACT 18 / CONCURRENT HOOKS

### 11. `useTransition`

**What:** Mark non-urgent updates
**Why:** Keep UI responsive
**How:** Low-priority state updates

```js
const [isPending, startTransition] = useTransition();
```

---

### 12. `useDeferredValue`

**What:** Deferred version of value
**Why:** Delay heavy rendering
**How:** Lags behind original value

```js
const deferredValue = useDeferredValue(value);
```

---

### 13. `useId`

**What:** Generate unique IDs
**Why:** SSR-safe IDs
**How:** Stable across server/client

```js
const id = useId();
```

---

### 14. `useSyncExternalStore`

**What:** Subscribe to external stores
**Why:** Correct concurrent rendering
**How:** Read from non-React stores

```js
useSyncExternalStore(subscribe, getSnapshot);
```

---

### 15. `useInsertionEffect`

**What:** Inject styles before layout
**Why:** CSS-in-JS libraries
**How:** Runs before layout effects

```js
useInsertionEffect(() => {}, []);
```

---

#### EXPERIMENTAL / ADVANCED (React 19+)

### 16. `useOptimistic`

**What:** Optimistic UI updates
**Why:** Instant feedback before server response
**How:** Temporary state updates

```js
const [state, addOptimistic] = useOptimistic(data);
```

---

### 17. `useActionState`

**What:** Track async action state
**Why:** Server actions handling
**How:** Handles pending/error/result

```js
const [state, action] = useActionState(fn, initial);
```

---

### 18. `useFormState`

**What:** Manage form submission state
**Why:** Server-driven forms
**How:** Sync with server actions

```js
const [state, formAction] = useFormState(action, {});
```

---

### 19. `useFormStatus`

**What:** Read form submission status
**Why:** Loading / disabled UI
**How:** Works inside `<form>`

```js
const { pending } = useFormStatus();
```

---

#### INTERVIEW ONE-LINER

> **React has 15 stable built-in hooks** and a few **experimental hooks** mainly used for forms and optimistic UI.
