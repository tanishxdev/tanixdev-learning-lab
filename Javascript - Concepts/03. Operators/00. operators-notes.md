# JavaScript Operators

### 1. What Are Operators in JavaScript?

Operators are special symbols or keywords that perform operations on operands (values/variables) and produce a result.

```js
let result = 10 + 5;   // + is the operator, 10 and 5 are operands
```

### 2. Operator Categories (Complete List)

| Category                  | Examples                                  | Purpose |
|---------------------------|-------------------------------------------|--------|
| Arithmetic                | `+ - * / % ** ++ --`                      | Math operations |
| Assignment                | `= += -= *= /= %= **= <<= >>= &= ^= |=`   | Assign values |
| Comparison                | `== != === !== > < >= <=`                 | Compare values |
| Logical                   | `&& || ! ?? &&= ||= ??=`                  | Boolean logic |
| Bitwise                   | `& | ^ ~ << >> >>>`                       | Binary-level manipulation |
| Ternary (Conditional)     | `condition ? expr1 : expr2`               | Short if-else |
| Optional Chaining         | `?.`                                      | Safe property access |
| Nullish Coalescing        | `??`                                      | Fallback for null/undefined |
| Comma                     | `,`                                       | Multiple expressions |
| typeof / instanceof / in  | `typeof`, `instanceof`, `in`              | Type/property checking |
| Spread / Rest             | `...`                                     | Expand / collect |
| Delete / void             | `delete`, `void`                          | Rarely used |

Now let’s go through each one in depth.

### 3. Operator Precedence & Associativity (Most Important Table)

| Precedence | Operator                              | Description                     | Associativity     |
|------------|---------------------------------------|---------------------------------|-------------------|
| 20         | `()`                                  | Grouping                        | n/a               |
| 19         | `. [] () ?. ??`                       | Member access, call, optional   | Left → Right      |
| 18         | `new` (with args)                     |                                 | n/a               |
| 17         | `++ --` (postfix)                     |                                 | n/a               |
| 16         | `++ -- ! ~ typeof void delete` (prefix) | Unary operators              | Right → Left      |
| 15         | `**`                                  | Exponentiation                  | Right → Left      |
| 14         | `* / %`                               | Multiply/Divide                 | Left → Right      |
| 13         | `+ -`                                 | Add/Subtract                    | Left → Right      |
| 12         | `<< >> >>>`                           | Bit shifts                      | Left → Right      |
| 11         | `< <= > >= in instanceof`             | Relational                      | Left → Right      |
| 10         | `== != === !==`                       | Equality                        | Left → Right      |
| 9          | `&`                                   | Bitwise AND                     | Left → Right      |
| 8          | `^`                                   | Bitwise XOR                     | Left → Right      |
| 7          | `|`                                   | Bitwise OR                      | Left → Right      |
| 6          | `&&`                                  | Logical AND                     | Left → Right      |
| 5          | `||`                                  | Logical OR                      | Left → Right      |
| 4          | `??`                                  | Nullish coalescing              | Left → Right      |
| 3          | `? :`                                 | Ternary                         | Right → Left      |
| 2          | Assignment operators (`= += &&= etc.`) |                         | Right → Left      |
| 1          | `,`                                   | Comma                           | Left → Right      |

Always use parentheses when in doubt – readability > cleverness.

### 4. Detailed Explanation with Real-World Code

#### 4.1 Arithmetic Operators

```js
let a = 10, b = 3;

// Basic
console.log(a + b);   // 13
console.log(a - b);   // 7
console.log(a * b);   // 30
console.log(a / b);   // 3.333...
console.log(a % b);   // 1  → remainder
console.log(a ** b);  // 1000 → 10^3

// Increment / Decrement
let count = 0;
console.log(++count);  // 1 (pre-increment)
console.log(count++);  // 1 (post-increment) → then count becomes 2
console.log(count);    // 2

// Unary + and -
let str = "50";
console.log(+str);    // 50  → fastest string → number conversion
console.log(-str);    // -50
```

Real use: `let total = price * quantity + tax;`

#### 4.2 Assignment Operators (Compound Assignment)

```js
let score = 100;
score += 20;    // score = score + 20 → 120
score -= 10;    // 110
score *= 2;     // 220
score /= 5;     // 44
score **= 2;    // 1936
```

Modern ones (ES2021):
```js
let x = 0;
x &&= 5;   // if x is truthy → assign 5, else do nothing → x remains 0
x ||= 10;  // if x is falsy → assign 10 → x becomes 10
let y = null;
y ??= 99;  // only assign if null or undefined → y = 99
```

#### 4.3 Comparison Operators – The Most Tricky Part

| Operator | Name               | Type Coercion? | Example             | Result |
|---------|--------------------|----------------|---------------------|--------|
| `==`    | Loose equality     | Yes            | `'5' == 5`          | true   |
| `===`   | Strict equality    | No             | `'5' === 5`         | false  |
| `!=`    | Loose inequality   | Yes            | `'5' != 5`          | false  |
| `!==`   | Strict inequality  | No             | `'5' !== 5`         | true   |

Never use `==` in production unless you have a very specific reason.

```js
console.log(null == undefined);  // true (special case)
console.log(null === undefined); // false

console.log(0 == false);         // true
console.log('' == false);        // true
console.log([] == false);        // true → [] → '' → 0 → false
```

Always prefer `===` and `!==`

#### 4.4 Logical Operators – Not Just Booleans!

```js
// Short-circuiting behavior (very powerful!)
let user = getUser();                 // might return null
let name = user && user.name;         // if user is null → name = null
let display = name || 'Guest';        // fallback

// Nullish coalescing (??) – only cares about null/undefined
let config = { timeout: 0 };
let time = config.timeout ?? 5000;    // 0 (not replaced!)
let time2 = config.timeout || 5000;   // 5000 (wrong! 0 is falsy)
```

Real-world pattern:
```js
const apiUrl = process.env.API_URL ?? 'https://api.default.com';
```

#### 4.5 Bitwise Operators (Used in performance-critical code)

```js
// Check if number is odd/even
if (num & 1) { /* odd */ }

// Swap two variables without temp
a = a ^ b;
b = a ^ b;
a = a ^ b;

// Color manipulation (RGBA)
let r = (color >> 24) & 255;
let g = (color >> 16) & 255;
let b = (color >> 8)  & 255;
let a = color & 255;
```

#### 4.6 Ternary Operator (Conditional)

```js
// Simple
const status = age >= 18 ? 'Adult' : 'Minor';

// Nested (avoid deep nesting!)
const role = isAdmin ? 'Admin' :
             isModerator ? 'Moderator' :
             isUser ? 'User' : 'Guest';

// With function calls
isLoggedIn ? renderDashboard() : renderLogin();
```

#### 4.7 Optional Chaining (?.)

```js
// No more "Cannot read property of undefined"
const city = user?.address?.city?.toLowerCase();
// Returns undefined instead of throwing error

// Optional chaining with function call
user?.onLogin?.();  // safe call
```

#### 4.8 typeof, instanceof, in

```js
typeof undefined     // "undefined"
typeof null          // "object" ← historical bug!
typeof []            // "object"
typeof function(){}  // "function"

Array.isArray(arr)   // proper way to check array

obj instanceof Map
'length' in arr      // true
```

### 5. Best Practices & Common Mistakes

| Mistake                              | Correct Way                              |
|--------------------------------------|------------------------------------------|
| Using `==` instead of `===`          | Always use `===`                         |
| `if (user == null)`                  | `if (user == null)` is okay, but `user ?? default` is better |
| Deeply nested ternary                | Use `if/else` or object lookup           |
| Forgetting optional chaining         | Always use `?.` when accessing nested props |
| Using `||` for default values when 0 or '' is valid | Use `??` instead |

### 6. One-Liner Summary for Interviews

"JavaScript operators are evaluated according to precedence (grouping highest, comma lowest). Always use strict equality (===), prefer ?? over || for defaults, use ?. to avoid null errors, and remember that logical operators return the last evaluated value (short-circuiting)."
