# 2. EACH COMPONENT DEEP UNDERSTANDING

---

## 2.1 JDK (Java Development Kit)

### Concept

- Used for **developing Java applications**
- Contains:
  - JRE
  - Tools like:
    - `javac` (compiler)
    - debugger
    - jar
    - javadoc

---

### Key Idea

**Developer uses JDK**

---

### What it does

1. Write code → `.java`
2. Compile using `javac`
3. Generate → `.class` (bytecode)

---

### Important Points

- Required for **coding**
- Platform dependent
- Includes everything (JRE + tools)

---

## 2.2 JRE (Java Runtime Environment)

### Concept

- Used for **running Java programs**
- Contains:
  - JVM
  - Libraries

---

### Key Idea

**User uses JRE to run app**

---

### What it does

1. Loads `.class`
2. Verifies bytecode
3. Runs using JVM

---

### Important Points

- No compiler
- Only runtime environment
- Platform dependent

---

## 2.3 JVM (Java Virtual Machine)

### Concept

- Core engine of Java
- Converts bytecode → machine code

---

### Key Idea

**Actual execution happens here**

---

### What it does

1. Load bytecode
2. Verify
3. Execute

---

### Internal Components

- Class Loader
- JIT Compiler
- Garbage Collector

---

### Important Points

- Platform dependent
- Makes Java platform independent

---

# 3. RELATIONSHIP (MOST IMPORTANT FOR INTERVIEW)

```
JDK = JRE + Development Tools
JRE = JVM + Libraries
JVM = Execution Engine
```

# 4. CODE FLOW (REAL EXECUTION FLOW)

---

```bash
javac Hello.java   // compile

```

---

## V2 — Full Example Program

```java
// File: Hello.java

// Main class
public class Hello {

    // Entry point of Java program
    public static void main(String[] args) {

        // Printing output
        System.out.println("Hello Java");

    }
}
```

---

# 5. DRY RUN / INTERNAL FLOW

Step-by-step:

1. You write:

   ```
   Hello.java
   ```

2. Compile:

   ```
   javac Hello.java
   ```

   → Converts into:

   ```
   Hello.class (bytecode)
   ```

3. Run:

   ```
   java Hello
   ```

4. JVM:
   - Loads class
   - Verifies
   - Converts to machine code
   - Executes

### Mistake 1

"JVM is platform independent"

Wrong

---

### Mistake 2

"JRE has compiler"

Wrong
→ Compiler only in JDK

---

### Mistake 3

"JDK needed to run program"

Not always
→ JRE enough to run
