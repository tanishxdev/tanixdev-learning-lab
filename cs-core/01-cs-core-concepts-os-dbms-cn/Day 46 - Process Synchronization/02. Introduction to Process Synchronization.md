## 1. Introduction

In a multitasking operating system, **multiple processes execute at the same time**.

Often, these processes:
- Share data
- Access common resources (memory, files, devices)
- Depend on each other’s execution

**Process Synchronization** is the mechanism used by the OS to **control and coordinate the execution of such processes** so that:

- Data remains consistent
- Conflicts are avoided
- The system behaves correctly

> In simple words:  
> **Process Synchronization ensures that multiple processes work together safely.**

---

## 2. Why Process Synchronization Exists (Problem First)

Consider this situation:

- Two processes update the same bank balance
- One adds ₹1000
- Another subtracts ₹500
- Both access the same memory location at the same time

### Without Synchronization

- Updates overlap
- Final balance becomes incorrect
- Data corruption occurs

This leads to **wrong results even though individual logic is correct**.

---

## 3. What Problems Occur Without Synchronization?

Improper or missing synchronization causes serious issues:

---

### 1. Data Inconsistency

- Multiple processes modify shared data simultaneously
- Updates overwrite each other
- Final data becomes unreliable

Example:
- Two processes increment the same variable
- Expected result = +2
- Actual result = +1 (one update lost)

---

### 2. Data Loss

- One process writes data
- Another process overwrites it before completion
- Important information is lost

---

### 3. Deadlock

- Processes wait forever for resources
- Each process holds one resource and waits for another

> Important clarification:  
> **Improper synchronization can lead to deadlock**, not mere absence of synchronization.

---

## 4. What is Process Synchronization?

### Definition

**Process Synchronization** is the coordination of **multiple cooperating processes** to ensure:

- Controlled access to shared resources
- Correct execution order
- Safe data sharing

---

## 5. Types of Processes (Based on Synchronization)

Processes are classified into two categories:

---

### 1. Independent Processes

- Execution of one process **does not affect** others
- No shared data
- No synchronization needed

Example:
- Calculator app
- Music player

---

### 2. Cooperative Processes

- Processes **affect or depend on each other**
- Share data or resources
- Synchronization is mandatory

Example:
- Producer–Consumer
- Compiler pipeline (ps | grep | wc)

---

## 6. Role of Synchronization in IPC

Process Synchronization plays a crucial role in **Inter-Process Communication (IPC)**:

- **Prevents Race Conditions**
- **Ensures Mutual Exclusion**
- **Coordinates Execution Order**
- **Avoids Deadlock**
- **Ensures Fairness (no starvation)**
- **Maintains data consistency**

---

## 7. Types of Process Synchronization

---

### 1. Competitive Synchronization

- Multiple processes **compete for the same resource**
- Only one should access it at a time

Example:
- Multiple processes writing to a file

**Risk without sync:**
- Data inconsistency
- Data loss

---

### 2. Cooperative Synchronization

- Processes **work together**
- Execution of one affects the other

Example:
```bash
ps | grep "chrome" | wc
````

* `ps` produces data
* `grep` filters it
* `wc` counts it

All processes depend on **correct execution order**.

**Risk without sync:**

* Deadlock
* Incorrect coordination

---

## 8. Conditions That Require Process Synchronization

---

### 1. Critical Section

A **critical section** is a part of code where:

* Shared variables are accessed
* Only **one process** must execute at a time

**Critical Section Problem**:
Design a solution so that:

* Only one process enters critical section
* Data remains consistent

---

### 2. Race Condition

A **race condition** occurs when:

* Output depends on execution order
* Multiple processes run concurrently in critical section

> Same code + different execution order = different result

---

### 3. Preemption

**Preemption** means:

* OS stops a running process
* CPU is given to another process

Problem:

* Process gets preempted **while updating shared data**
* Another process reads inconsistent data

Hence, synchronization is required.

## 10. Goals of Process Synchronization

A good synchronization mechanism must ensure:

1. **Mutual Exclusion**
   Only one process in critical section

2. **Progress**
   No unnecessary blocking

3. **Bounded Waiting**
   No process waits forever (no starvation)

## 12. Key Takeaways

* Process synchronization ensures **correct cooperation**
* Prevents:

  * Race conditions
  * Data inconsistency
  * Deadlocks
* Critical section is the core problem
* Foundation for:

  * Semaphores
  * Mutex
  * Monitors
