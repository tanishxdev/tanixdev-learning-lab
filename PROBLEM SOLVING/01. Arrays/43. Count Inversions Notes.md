# PROBLEM (Original Statement)

Given an array of integers `nums`, return the number of inversions in the array.

An inversion is defined as a pair `(i, j)` such that:

- `i < j`
- `nums[i] > nums[j]`

---

## Topics / Patterns

**Topic:**

- Arrays
- Divide and Conquer

**Patterns:**

1. Brute Force (Double Loop)
2. Merge Sort Modification
3. Counting During Merge Phase

This is a classic:

> “Count something while sorting”

Whenever you see:

- Condition: `i < j`
- Comparison across left and right
- Need efficient counting

Think:

→ Merge Sort Trick

---

## BREAKDOWN PROBLEM

We need to count how many pairs `(i, j)` exist where:

- i comes before j
- Value at i is greater than value at j

Example:

[2, 4, 1, 3, 5]

Inversions:
(2,1)
(4,1)
(4,3)

Total = 3

---

## CONSTRAINT UNDERSTANDING

Typical constraints:

- 1 ≤ n ≤ 10⁵
- -10⁹ ≤ nums[i] ≤ 10⁹

Important:

If n = 10⁵,

Brute O(n²) → 10¹⁰ comparisons → impossible.

We need:

O(n log n)

---

# BRUTE FORCE APPROACH

## First Thought Intuition

Natural thought:

Check every pair.

Because definition directly says:
Check all `(i, j)` where `i < j`.

---

## Thought Process (Step-wise)

1. Loop i from 0 to n-1
2. Loop j from i+1 to n-1
3. If nums[i] > nums[j], increment count

---

## Pseudocode

```
count = 0
for i from 0 to n-1:
    for j from i+1 to n-1:
        if nums[i] > nums[j]:
            count++

return count
```

---

## Algorithm

Just nested loops and comparison.

---

## CODE

### C++

#### V1 (Only Required Function)

```cpp
long long countInversions(vector<int>& nums) {

    long long count = 0;

    int n = nums.size();

    for (int i = 0; i < n; i++) {

        for (int j = i + 1; j < n; j++) {

            if (nums[i] > nums[j]) {
                count++;
            }
        }
    }

    return count;
}
```

# BETTER APPROACH

Now we think deeper.

Observation:

Inversion is about:

Element in left side greater than element in right side.

This reminds of:

Merge Sort.

Why?

Because merge sort splits array into left and right halves.

During merge:

- If left[i] > right[j]
  → then ALL remaining elements in left after i are also > right[j]

This allows bulk counting.

This is the key insight.

---

# OPTIMAL APPROACH

## First Thought Intuition

Modify merge sort.

While merging:

If left[i] > right[j]
→ count += (mid - i + 1)

Because all remaining elements in left half are greater.

---

## Thought Process (Step-wise)

1. Divide array into halves.
2. Recursively count inversions in left.
3. Recursively count inversions in right.
4. Count cross inversions during merge.

Total inversions =
left inversions +
right inversions +
merge inversions

---

## Why This Works

Because:

Both halves are sorted before merging.

So:

If left[i] > right[j]

Then:
left[i+1], left[i+2], ..., left[mid]
are all greater than right[j].

So we count them in one go.

---

## Pseudocode

```
function mergeSort(arr, low, high):

    if low >= high:
        return 0

    mid = (low + high) / 2

    leftCount = mergeSort(arr, low, mid)
    rightCount = mergeSort(arr, mid+1, high)
    mergeCount = merge(arr, low, mid, high)

    return leftCount + rightCount + mergeCount
```

Merge:

```
i = low
j = mid+1

while i <= mid and j <= high:

    if arr[i] <= arr[j]:
        i++
    else:
        count += (mid - i + 1)
        j++
```

---

## CODE

### C++

#### V1 (Only Required Function)

```cpp
long long mergeAndCount(vector<int>& nums, int low, int mid, int high) {

    vector<int> temp;

    int i = low;
    int j = mid + 1;

    long long count = 0;

    while (i <= mid && j <= high) {

        if (nums[i] <= nums[j]) {

            temp.push_back(nums[i]);
            i++;
        }
        else {

            // nums[i] > nums[j] → inversion found
            count += (mid - i + 1);

            temp.push_back(nums[j]);
            j++;
        }
    }

    while (i <= mid) {
        temp.push_back(nums[i]);
        i++;
    }

    while (j <= high) {
        temp.push_back(nums[j]);
        j++;
    }

    for (int k = 0; k < temp.size(); k++) {
        nums[low + k] = temp[k];
    }

    return count;
}

long long mergeSort(vector<int>& nums, int low, int high) {

    if (low >= high)
        return 0;

    int mid = low + (high - low) / 2;

    long long count = 0;

    count += mergeSort(nums, low, mid);
    count += mergeSort(nums, mid + 1, high);
    count += mergeAndCount(nums, low, mid, high);

    return count;
}

long long countInversions(vector<int>& nums) {

    return mergeSort(nums, 0, nums.size() - 1);
}
```

#### V2 (Complete Program)

```cpp
#include <iostream>
#include <vector>
using namespace std;

long long mergeAndCount(vector<int>& nums, int low, int mid, int high) {

    vector<int> temp;

    int i = low;
    int j = mid + 1;

    long long count = 0;

    while (i <= mid && j <= high) {

        if (nums[i] <= nums[j]) {

            temp.push_back(nums[i]);
            i++;
        }
        else {

            // All remaining elements in left half form inversion
            count += (mid - i + 1);

            temp.push_back(nums[j]);
            j++;
        }
    }

    while (i <= mid) {
        temp.push_back(nums[i]);
        i++;
    }

    while (j <= high) {
        temp.push_back(nums[j]);
        j++;
    }

    for (int k = 0; k < temp.size(); k++) {
        nums[low + k] = temp[k];
    }

    return count;
}

long long mergeSort(vector<int>& nums, int low, int high) {

    if (low >= high)
        return 0;

    int mid = low + (high - low) / 2;

    long long count = 0;

    count += mergeSort(nums, low, mid);
    count += mergeSort(nums, mid + 1, high);
    count += mergeAndCount(nums, low, mid, high);

    return count;
}

long long countInversions(vector<int>& nums) {

    return mergeSort(nums, 0, nums.size() - 1);
}

int main() {

    vector<int> nums = {2, 4, 1, 3, 5};

    cout << "Inversions: " << countInversions(nums);

    return 0;
}
```
