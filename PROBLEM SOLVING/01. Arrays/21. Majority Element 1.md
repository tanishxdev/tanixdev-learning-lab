# PROBLEM (Original Statement)

Given an integer array `nums` of size `n`, find the **majority element**.

A majority element is the element that appears **more than ⌊ n/2 ⌋ times**.

You may assume that **the majority element always exists**.

Example

```
nums = [3,2,3]
Output = 3
```

```
nums = [2,2,1,1,1,2,2]
Output = 2
```

---

| Brute    | Nested counting                  |
| Better   | Hash Map / Frequency             |
| Optimal  | **Boyer–Moore Voting Algorithm** |

Why Boyer–Moore works:

```
Majority element count > n/2
```

So it **cannot be cancelled completely**.

# BREAKDOWN PROBLEM

Input

```
nums = [2,2,1,1,1,2,2]
```

n = 7

Majority condition

```
count > floor(7/2)
count > 3
```

2 occurs **4 times → majority element**

# CONSTRAINT UNDERSTANDING

Typical constraints

```
1 ≤ n ≤ 10^5
```

If we do nested loops:

```
O(n²)
```

Too slow for large `n`.

We want **O(n)**.

---

# BRUTE FORCE APPROACH

## First Thought Intuition

Check the frequency of each element.

If frequency > n/2 → answer.

---

## Thought Process

For each element

1. Count how many times it appears.
2. If count > n/2 return element.

---

## Pseudocode

```
for i from 0 to n-1

    count = 0

    for j from 0 to n-1
        if nums[j] == nums[i]
            count++

    if count > n/2
        return nums[i]
```

---

## CODE

### C++ V1 (Function Only)

```cpp
int majorityElement(vector<int>& nums)
{
    int n = nums.size();

    for(int i=0;i<n;i++)
    {
        int count = 0;

        for(int j=0;j<n;j++)
        {
            if(nums[j] == nums[i])
                count++;
        }

        if(count > n/2)
            return nums[i];
    }

    return -1;
}
```

Pattern

# First Thought Intuition

Instead of counting repeatedly,

store frequency in a **map**.

---

## Thought Process

Traverse array

```
map[num]++
```

Check

```
if map[num] > n/2
```

Return.

---

## Pseudocode

```
create hashmap

for num in nums
    freq[num]++

for each element
    if freq > n/2
        return element
```

---

## CODE

### C++ V1

```cpp
int majorityElement(vector<int>& nums)
{
    unordered_map<int,int> mp;

    for(int x : nums)
        mp[x]++;

    int n = nums.size();

    for(auto it : mp)
    {
        if(it.second > n/2)
            return it.first;
    }

    return -1;
}
```

# OPTIMAL APPROACH

Pattern

```
Boyer–Moore Voting Algorithm
```

This is one of the **most important interview algorithms**.

---

# First Thought Intuition

If majority element appears **more than half**, it **cannot be cancelled out**.

Example

```
[2,2,1,1,1,2,2]
```

Cancel pairs

```
2 vs 1 cancel
2 vs 1 cancel
2 remains
```

Majority survives.

---

# Thought Process

Maintain

```
candidate
count
```

Rules

```
if count == 0
    candidate = nums[i]

if nums[i] == candidate
    count++

else
    count--
```

---

# Pseudocode

```
candidate = none
count = 0

for num in nums

    if count == 0
        candidate = num

    if num == candidate
        count++

    else
        count--
```

Return candidate.

---

# CODE

### C++ V1

```cpp
int majorityElement(vector<int>& nums)
{
    int count = 0;
    int candidate = 0;

    for(int num : nums)
    {
        if(count == 0)
            candidate = num;

        if(num == candidate)
            count++;
        else
            count--;
    }

    return candidate;
}
```
