# PROBLEM (Original Statement)

Given an integer array `nums`, find the **contiguous subarray** that has the **largest sum**, and return that sum.

Example

```
nums = [-2,1,-3,4,-1,2,1,-5,4]
Output = 6
```

Explanation

```
Subarray = [4,-1,2,1]
Sum = 6
```

---

# Topics / Patterns

### Topic

Array

### Patterns

| Approach | Pattern                |
| -------- | ---------------------- |
| Brute    | Generate all subarrays |
| Better   | Running sum            |
| Optimal  | **Kadane's Algorithm** |

Why Kadane works:

```
If current sum becomes negative,
it cannot help future subarrays.
```

So we reset.

---

# BREAKDOWN PROBLEM

Input

```
nums = [-2,1,-3,4,-1,2,1,-5,4]
```

Possible subarrays

```
[-2]
[-2,1]
[-2,1,-3]
[1]
[1,-3]
...
[4,-1,2,1]
```

We must find **maximum sum among all contiguous subarrays**.

---

# CONSTRAINT UNDERSTANDING

Typical constraints

```
1 ≤ n ≤ 10^5
```

Total possible subarrays

```
n(n+1)/2
```

For `n = 10^5`

```
≈ 5 × 10^10
```

Too large → brute force is slow.

---

# BRUTE FORCE APPROACH

## First Thought Intuition

Generate every subarray.

Compute its sum.

Track maximum.

---

## Thought Process

1. Start from index `i`
2. Extend subarray to `j`
3. Compute sum
4. Update maximum

---

## Pseudocode

```
maxSum = -∞

for i from 0 to n-1

    sum = 0

    for j from i to n-1

        sum += nums[j]

        maxSum = max(maxSum,sum)
```

---

## CODE

### C++ V1 (Function Only)

```cpp
int maxSubArray(vector<int>& nums)
{
    int n = nums.size();
    int maxSum = INT_MIN;

    for(int i=0;i<n;i++)
    {
        int sum = 0;

        for(int j=i;j<n;j++)
        {
            sum += nums[j];
            maxSum = max(maxSum,sum);
        }
    }

    return maxSum;
}
```

---

# BETTER APPROACH

Observation

We do not need to recompute sums repeatedly.

We can maintain **running sum**.

This reduces work but still `O(n²)`.

(Already implemented in brute optimized form above.)

---

# OPTIMAL APPROACH

Pattern

```
Kadane's Algorithm
```

---

# First Thought Intuition

If the current sum becomes **negative**, it will only reduce future sums.

Example

```
current sum = -5
next number = 4
```

Subarray sum

```
-5 + 4 = -1
```

Better to start fresh:

```
4
```

So we **reset sum when it becomes negative**.

---

# Thought Process

Maintain

```
currentSum
maxSum
```

For each element

```
currentSum += nums[i]

maxSum = max(maxSum,currentSum)

if currentSum < 0
    reset currentSum = 0
```

---

# Pseudocode

```
maxSum = -∞
currentSum = 0

for num in nums

    currentSum += num

    maxSum = max(maxSum,currentSum)

    if currentSum < 0
        currentSum = 0
```

---

# CODE

### C++ V1 (Function Only)

```cpp
int maxSubArray(vector<int>& nums)
{
    int maxSum = INT_MIN;
    int currentSum = 0;

    for(int num : nums)
    {
        currentSum += num;

        maxSum = max(maxSum,currentSum);

        if(currentSum < 0)
            currentSum = 0;
    }

    return maxSum;
}
```

---

### C++ V2 (Complete Program)

```cpp
#include <iostream>
#include <vector>

int main()
```
