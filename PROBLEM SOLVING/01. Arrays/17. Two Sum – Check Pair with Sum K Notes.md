# PROBLEM (Original Statement)

Given an integer array `nums` and an integer `target`, determine whether there exist **two numbers in the array whose sum equals the target**.

Two variants are considered:

**Variant 1**

Return `"YES"` if such a pair exists, otherwise return `"NO"`.

**Variant 2**

Return the **indices of the two elements** whose sum equals the target.
If no such pair exists, return `{-1, -1}`.

---

# Topics / Patterns

### Topic

Array

Why:
The problem directly involves scanning and processing elements of an array.

### Patterns

**1. Brute Pattern → Nested Loop / Pair Enumeration**

Check every possible pair `(i, j)`.

Why:
This is the most natural first thinking when we need to check combinations.

---

**2. Better Pattern → Sorting + Two Pointer**

Sort the array and move two pointers.

Why:
When array is sorted, sum comparisons guide pointer movement.

---

**3. Optimal Pattern → Hashing**

Use a hash map to store seen numbers.

Why:
Allows constant-time lookup for `target - current`.

```

```

```

### Output

```

# CONSTRAINT UNDERSTANDING

Typical constraints:

```
1 ≤ n ≤ 10^5
-10^9 ≤ nums[i] ≤ 10^9
```

Implications:

| Approach    | Time       |
| ----------- | ---------- |
| Brute       | O(n²)      |
| Two Pointer | O(n log n) |
| Hashing     | O(n)       |

For large `n`, **O(n²) is too slow**, so hashing becomes optimal.

# BRUTE FORCE APPROACH

## First Thought Intuition

First natural thought:

> Check every pair of numbers.

Because if we test every combination, we cannot miss the answer.

---

## Thought Process (Step-wise)

1. Take first element.
2. Pair it with every other element.
3. Check if sum equals target.
4. If yes → return result.
5. If no pair found → return failure.

---

## Pseudocode

```
for i from 0 to n-1
    for j from i+1 to n-1
        if nums[i] + nums[j] == target
            return answer
return failure
```

---

## Algorithm

1. Loop through array using `i`
2. Inner loop checks `j`
3. Compare sum
4. Return accordingly

# Variant 1 → Return "YES" / "NO"

### V1 (Only Required Function)

```cpp
string twoSumExists(vector<int>& nums, int target)
{
    int n = nums.size();

    // Check every pair
    for(int i = 0; i < n; i++)
    {
        for(int j = i + 1; j < n; j++)
        {
            // If pair sum equals target
            if(nums[i] + nums[j] == target)
            {
                return "YES";
            }
        }
    }

    return "NO";
}
```

# BETTER APPROACH

## Idea

Use **sorting + two pointer technique**.

---

# First Thought Intuition

If the array were **sorted**, then:

- If sum is **too small** → move left pointer right
- If sum is **too big** → move right pointer left

This avoids checking every pair.

---

# Thought Process

Example

```
nums = [2,7,11,15]
target = 9
```

Sorted:

```
[2,7,11,15]
```

Use two pointers

```
L = 0
R = n-1
```

Check:

```
nums[L] + nums[R]
```

Cases

```
sum < target → move L++
sum > target → move R--
sum == target → answer found
```

---

# Important Note

Sorting **changes indices**.

So we must store:

```
(value, originalIndex)
```

---

# Pseudocode

```
store pair(value,index)

sort array

left = 0
right = n-1

while left < right
    sum = arr[left] + arr[right]

    if sum == target
        return answer

    if sum < target
        left++

    else
        right--
```

---

# Algorithm

1. Store `(value,index)`
2. Sort array
3. Use two pointers
4. Move pointers based on sum

# CODE

## Variant 1 (YES / NO)

### V1 (Function Only)

```cpp
string twoSumTwoPointer(vector<int>& nums, int target)
{
    int n = nums.size();

    sort(nums.begin(), nums.end());

    int left = 0;
    int right = n - 1;

    while(left < right)
    {
        int sum = nums[left] + nums[right];

        if(sum == target)
            return "YES";

        if(sum < target)
            left++;
        else
            right--;
    }

    return "NO";
}
```

## Variant 2 (Return Indices)

### V1 (Function Only)

```cpp
vector<int> twoSumTwoPointerIndex(vector<int>& nums, int target)
{
    int n = nums.size();

    vector<pair<int,int>> arr;

    for(int i=0;i<n;i++)
        arr.push_back({nums[i], i});

    sort(arr.begin(), arr.end());

    int left = 0;
    int right = n-1;

    while(left < right)
    {
        int sum = arr[left].first + arr[right].first;

        if(sum == target)
            return {arr[left].second, arr[right].second};

        if(sum < target)
            left++;
        else
            right--;
    }

    return {-1,-1};
}
```

# OPTIMAL APPROACH

Pattern:

```
Hash Map
```

---

# First Thought Intuition

Instead of searching pairs repeatedly:

Store **numbers we have already seen**.

For every element:

```
required = target - nums[i]
```

Check:

```
Have we seen required before?
```

If yes → pair found.

---

# Thought Process

Example

```
nums = [2,7,11,15]
target = 9
```

Step 1

```
i=0
num=2
need=7
map={}
```

store

```
map[2] = 0
```

---

Step 2

```
i=1
num=7
need=2
```

map contains 2

Answer found.

---

# Pseudocode

```
create hashmap

for i from 0 to n-1

    need = target - nums[i]

    if need exists in map
        return answer

    store nums[i] in map
```

---

# Algorithm

1. Traverse array once
2. Compute complement
3. Lookup in hashmap
4. Return if found

# CODE

## Variant 1 (YES / NO)

### V1 (Function Only)

```cpp
string twoSumHash(vector<int>& nums, int target)
{
    unordered_map<int,int> mp;

    for(int i=0;i<nums.size();i++)
    {
        int need = target - nums[i];

        if(mp.find(need) != mp.end())
            return "YES";

        mp[nums[i]] = i;
    }

    return "NO";
}
```
