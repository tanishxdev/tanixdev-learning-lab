

# **SET vs HASH SET (unordered_set)**

## 1. **Definition**

| Type              | Name        | Internally Uses               |
| ----------------- | ----------- | ----------------------------- |
| **set**           | Ordered Set | Balanced BST (Red-Black Tree) |
| **unordered_set** | Hash Set    | Hash Table                    |

---

# 2. **Sorting / Ordering**

| Feature                | set | unordered_set |
| ---------------------- | --- | ------------- |
| Stored in sorted order | Yes | No            |
| Order predictable      | Yes | No            |

**Example:**

```
set<int> s = {5,2,1};
Output: 1 2 5

unordered_set<int> s = {5,2,1};
Output: random order (e.g., 2 5 1)
```

---

# 3. **Time Complexity**

| Operation | set      | unordered_set |
| --------- | -------- | ------------- |
| Insertion | O(log n) | O(1) average  |
| Search    | O(log n) | O(1) average  |
| Delete    | O(log n) | O(1) average  |

unordered_set is **faster** in most cases.

---

# 4. **Duplicates**

Both remove duplicates automatically.

---

# 5. **When to Use What?**

## **Use set when:**

1. Sorted output chahiye
2. Range queries karne ho (lower_bound, upper_bound)
3. Ordered traversal chahiye

## **Use unordered_set when:**

1. Fastest lookup/insertion chahiye
2. Order matter nahi karta
3. Only membership check required (x present or not)

Example:

```
if(x exists?)  -> unordered_set best
```

---

# 6. **Why UNION uses set sometimes?**

Because union ka output sorted chahiye hota hai (TUF, GFG style).
set gives sorted + unique automatically.

unordered_set gives unique but **not sorted**, so extra sorting needed.

---

# 7. **Why INTERSECTION uses unordered_set?**

Because intersection me bas check karna hota hai:

```
x in set ?
```

unordered_set gives O(1) membership â†’ fastest.

Sorted output required ho to sort at end.

---

# 8. **Memory**

unordered_set thoda extra memory leta hai (hash table buckets).

---

# **Short Summary (Most Important)**

```
set = slow (log n), sorted, uses BST
unordered_set = fast (O(1)), not sorted, uses hash table
```
