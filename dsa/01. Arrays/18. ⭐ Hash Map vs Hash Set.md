
# Hash Map vs Hash Set

---

## 1. What is Hashing?

Hashing is a technique used to store and find data in **O(1) average time** using a **hash function**.
It maps a key (number/string) to a bucket (index).

Example idea:

```
key = 10  
hash(10) → index 4 in memory
```

---

# 2. Hash Set (`unordered_set`)

A Hash Set stores **unique values only**.
There are no duplicate elements.

You only check if a value exists or not.

### When to use Hash Set?

* Check if a number already exists
* Remove duplicates
* Union of elements
* Detect cycle or repeating values
* Fast membership test: `if(set.count(x))`

### Basic operations

```cpp
unordered_set<int> s;

s.insert(5);
s.insert(10);

if(s.count(10)) {
    cout << "10 exists";
}

s.erase(5);
```

### Time Complexity

| Operation | TC (Average) |
| --------- | ------------ |
| insert(x) | O(1)         |
| find(x)   | O(1)         |
| erase(x)  | O(1)         |

### Example Use Cases in your array topics:

| Problem                             | Technique                       |
| ----------------------------------- | ------------------------------- |
| Union of two arrays                 | use set to keep unique elements |
| Remove duplicates from sorted array | store seen values               |
| Check if pair sum exists            | seen set                        |
| Find duplicate                      | insert and check exists         |
| Cycle detection                     | store visited items             |

---

# 3. Hash Map (`unordered_map`)

A Hash Map stores **key → value** pairs.

Example:

```
Key = number
Value = frequency or index
```

### When to use Hash Map?

* Counting frequency of elements
* Two Sum (store index of required value)
* Find number appearing once (freq checking)
* Prefix Sum (store sum → index)
* Longest Subarray with sum K

### Basic operations

```cpp
unordered_map<int, int> freq;

freq[10] = 1;
freq[20] = 5;

if(freq.count(10)) {
    cout << "Key exists";
}

freq.erase(20);
```

### Time Complexity

| Operation | TC (Average) |
| --------- | ------------ |
| insert    | O(1)         |
| find      | O(1)         |
| erase     | O(1)         |
| access    | O(1)         |

### Example Use Cases in your array folder:

| Problem                           | Use of Hash Map      |
| --------------------------------- | -------------------- |
| Two Sum                           | store number → index |
| Single Number (once, other twice) | count freq           |
| Missing Number                    | mark visited         |
| Longest Subarray Sum K            | prefix sum → index   |
| Intersection with counts          | store frequencies    |

---

# 4. Difference Summary Table

| Feature      | Hash Set (`unordered_set`)        | Hash Map (`unordered_map`)   |
| ------------ | --------------------------------- | ---------------------------- |
| Stores       | only values                       | key → value pairs            |
| Duplicates   | no                                | key unique, value may repeat |
| Use case     | membership test                   | frequency, indexing          |
| Example      | `{1,4,5}`                         | `{1:2, 4:1, 5:3}`            |
| Two Sum role | sometimes check complement exists | store complement's index     |
| Prefix role  | rarely used                       | stores prefixSum → index     |
| Memory       | lower generally                   | depends on values stored     |

# 5. Code Comparison Example

### Using Hash Set

Check if array contains duplicates.

```cpp
bool hasDuplicate(vector<int>& arr) {
    unordered_set<int> s;
    for(int x : arr) {
        if(s.count(x)) return true;
        s.insert(x);
    }
    return false;
}
```

### Using Hash Map

Count frequency of each element.

```cpp
unordered_map<int,int> freq;
for(int x : arr) freq[x]++;

for(auto &p : freq) {
    cout << p.first << " occurs " << p.second << " times" << endl;
}
```

# 6. When to choose which?

| Goal                          | Choose   |
| ----------------------------- | -------- |
| Need to check if value exists | Hash Set |
| Need to remove duplicates     | Hash Set |
| Need to count frequency       | Hash Map |
| Need index of seen value      | Hash Map |
| Need prefix sum storage       | Hash Map |
| Need Two Sum indices          | Hash Map |
| Need unique elements only     | Hash Set |

---

# 7. Memory Notes

* Hash Set: stores only values → lighter
* Hash Map: stores both key & value → more memory

---

# Final Short Summary (Fast Revision)

| Concept  | One-line meaning                                              |
| -------- | ------------------------------------------------------------- |
| Hash Set | stores unique values, fast existence check                    |
| Hash Map | stores key-value pairs, used for frequency, index, prefix sum |
