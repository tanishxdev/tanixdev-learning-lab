
# HASHING CHEAT SHEET (Quick Revision)

---

## 1. What is Hashing?

A technique to store and find values in **average O(1)** time using a **hash function**.

Examples where hashing helps:

* Find if value exists
* Count frequency
* Check duplicates
* Searching in O(1)
* Prefix sum storage

---

## 2. Hash Set (`unordered_set`)

Stores unique values only.

### Syntax

```cpp
unordered_set<int> s;
s.insert(x);
s.erase(x);
if(s.count(x)) { /* exists */ }
```

### Time Complexity

| Operation | Avg TC |
| --------- | ------ |
| insert(x) | O(1)   |
| count(x)  | O(1)   |
| erase(x)  | O(1)   |

### When to use

Use Hash Set when you only need:

* element existence check
* remove duplicates
* track visited elements

### Common Problems (Arrays Folder)

| Problem            | How Hash Set helps    |
| ------------------ | --------------------- |
| Union of arrays    | store unique numbers  |
| Intersection       | check existence       |
| Contains Duplicate | check seen values     |
| Two Sum (YES/NO)   | check target-x exists |

---

## 3. Hash Map (`unordered_map`)

Stores key → value pairs. Useful for indexing, counting.

### Syntax

```cpp
unordered_map<int,int> mp;
mp[x]++;          // count frequency
mp[x] = index;    // store position
mp.count(x);      // check existence
mp.erase(x);
```

### Time Complexity

| Operation  | Avg TC |
| ---------- | ------ |
| insert     | O(1)   |
| find/count | O(1)   |
| erase      | O(1)   |
| access     | O(1)   |

### When to use

Use Hash Map when you need:

* frequency count
* index lookup
* prefix sum storage
* value-to-index mapping (Two Sum)

### Common Problems (Arrays Folder)

| Problem                      | Why Hash Map               |
| ---------------------------- | -------------------------- |
| Two Sum (with indices)       | store number → index       |
| Single number (others twice) | count frequency            |
| Missing number               | mark visited               |
| Longest Subarray Sum K       | prefixSum → earliest index |

---

## 4. Hash Map vs Hash Set (Comparison)

| Feature            | Hash Set (`unordered_set`)           | Hash Map (`unordered_map`)      |
| ------------------ | ------------------------------------ | ------------------------------- |
| Stores             | only values                          | key → value pairs               |
| Duplicates allowed | no                                   | keys unique, values may repeat  |
| Use case           | existence check                      | frequency/index mapping         |
| Example            | {2,5,7}                              | {2:1, 5:3, 7:2}                 |
| Used in            | union, intersection, duplicate check | two sum, prefix sum, freq count |

## 5. Hashing in Two Sum

### YES/NO version

```cpp
unordered_set<int> seen;
for(int x : arr){
    if(seen.count(target - x)) return "YES";
    seen.insert(x);
}
return "NO";
```

### Index version

```cpp
unordered_map<int,int> mp;
for(int i=0;i<n;i++){
    int need = target - arr[i];
    if(mp.count(need)) return {mp[need], i};
    mp[arr[i]] = i;
}
```

## 6. Hashing in Prefix Sum (Most Important for Subarray Problems)

Check if subarray sum equals k.

Key idea:
If prefix[i] - k == prefix[j],
subarray j+1..i has sum k.

```cpp
unordered_map<int,int> mp;
int sum = 0, maxLen = 0;

for(int i=0;i<n;i++){
    sum += arr[i];

    if(sum == k) maxLen = i+1;

    if(mp.count(sum - k))
        maxLen = max(maxLen, i - mp[sum-k]);

    if(!mp.count(sum))
        mp[sum] = i;
}
```

## 7. Common Mistakes in Hashing

| Mistake                            | Correction                  |
| ---------------------------------- | --------------------------- |
| Using map instead of unordered_map | map is slower (O(log n))    |
| Inserting prefix sum every time    | store only first occurrence |
| Using set when frequency needed    | use map instead             |
| For sorted two pointer problems    | hashing not required        |
| Expecting O(1) worst case          | worst case can be O(n)      |

---

## 8. Practice Patterns Summary

| Pattern                 | Data structure    |
| ----------------------- | ----------------- |
| Check duplicate         | Hash Set          |
| Two Sum index           | Hash Map          |
| Frequency Problems      | Hash Map          |
| Missing Number          | Set / Map / XOR   |
| Union of arrays         | Hash Set          |
| Intersection            | Hash Set or Map   |
| Longest Subarray Sum K  | Prefix + Hash Map |
| Single element in pairs | Hash Map or XOR   |

---

## 9. Quick Takeaways

* Hash Set: for membership test
* Hash Map: for value→frequency or value→index
* Best for O(1) lookup
* Prefix + Hash Map solves subarray sum problems
* Used heavily in array, string, and graph questions
