# SECTION B
## LLD Questions

• **Each of the 35 LLD questions** has:

* Expanded **problem statement**
* Detailed **what exactly needs to be designed**
* **Functional requirements** (must-have features)
* **Non-functional requirements** (performance, scalability, etc.)
* **Key challenges** (why this design is hard)
* **What interviewers expect from you**
* **Which LLD pattern(s)** are naturally used in this problem
* **How the system should behave** (high-level)

## 1. Tic-Tac-Toe

**Problem Statement:**
Design a two-player grid-based game where users mark X/O on a 3×3 board. Determine win/draw and manage turns.

**Functional Requirements:**
• Players take turns
• Validate moves
• Detect win/draw
• Reset game

**Non-Functional Requirements:**
• Clear separation between game logic and UI

**Challenges:**
• Efficient win detection
• Representing board state cleanly

**Interview Expectations:**
• Basic OOP
• Clean GameController + Board

**Patterns:** None (pure OOP)
