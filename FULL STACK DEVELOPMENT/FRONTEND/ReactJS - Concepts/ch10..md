# 43. Hooks Introduction

Tag: **[BOTH]**

This is one of the **most important concepts in modern React** because today **almost all React applications use Hooks instead of class components**.

Before learning `useState`, `useEffect`, etc., we must clearly understand **why Hooks were introduced**.

---

# 1. The Problem (Why Hooks Exist)

Early React (before 2019) used **Class Components** for state and lifecycle logic.

Example:

```jsx
class Counter extends React.Component {
  constructor() {
    super();
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <div>
        <h1>{this.state.count}</h1>
        <button onClick={this.increment}>Increase</button>
      </div>
    );
  }
}
```

Problems with class components:

### 1. Classes were complex

You had to understand:

- `this`
- constructors
- binding methods
- lifecycle methods

Example problem:

```js
this.setState(...)
```

Many beginners struggled with **`this` behavior**.

---

### 2. Logic was scattered across lifecycle methods

Example:

Data fetching required:

```
componentDidMount
componentDidUpdate
componentWillUnmount
```

Code related to **one feature** was spread across multiple places.

Example:

```
componentDidMount -> fetch API
componentDidUpdate -> refetch
componentWillUnmount -> cleanup
```

Hard to maintain.

---

### 3. Code reuse was difficult

To reuse logic developers used:

- Higher Order Components (HOC)
- Render Props

Both made code **deeply nested and confusing**.

Example structure:

```
<App>
   <AuthProvider>
      <ThemeProvider>
         <DataProvider>
            <Component />
```

This problem is called:

```
Wrapper Hell
```

---

# 2. The Solution: Hooks

React introduced **Hooks in React 16.8**.

Hooks allow **functional components to use state and lifecycle features**.

Before Hooks:

```
Class components → state + lifecycle
Functional components → UI only
```

After Hooks:

```
Functional components → UI + state + lifecycle
```

Example with Hooks:

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}
```

Much simpler.

---

# 3. What a Hook Actually Is

A **Hook is a special function provided by React that lets you use React features inside functional components.**

Example Hooks:

```
useState
useEffect
useContext
useReducer
useRef
useMemo
useCallback
```

All hooks follow a naming rule:

```
useSomething
```

Examples:

```
useState()
useEffect()
useContext()
```

---

# 4. Important Hook Rules

React hooks follow **strict rules**.

### Rule 1

Hooks must be called **at the top level of the component**.

Correct:

```jsx
function App() {
  const [count, setCount] = useState(0);
}
```

Wrong:

```jsx
if (condition) {
  useState(0);
}
```

Reason:

React tracks hooks using **call order**.

---

### Rule 2

Hooks only work inside:

```
React functional components
OR
custom hooks
```

Wrong:

```js
function normalFunction() {
  useState(0);
}
```

Correct:

```jsx
function Component() {
  const [value, setValue] = useState(0);
}
```

---

# 5. Hooks Replace Class Features

Hooks replaced these class features.

| Class Feature     | Hook Replacement |
| ----------------- | ---------------- |
| state             | useState         |
| lifecycle methods | useEffect        |
| complex state     | useReducer       |
| context access    | useContext       |
| DOM access        | useRef           |

Example comparison.

---

### Class Component

```jsx
class App extends React.Component {
  componentDidMount() {
    console.log("mounted");
  }

  render() {
    return <div>Hello</div>;
  }
}
```

---

### Hook Version

```jsx
import { useEffect } from "react";

function App() {
  useEffect(() => {
    console.log("mounted");
  }, []);

  return <div>Hello</div>;
}
```

Much simpler.

---

# 7. Real Example (Small App)

Example: counter + title update.

```jsx
import { useState, useEffect } from "react";

function Counter() {
  // state variable
  const [count, setCount] = useState(0);

  // lifecycle logic
  useEffect(() => {
    document.title = `Count is ${count}`;

    <div>

    </div>

}

export default Counter;
```

What happens:

```
count changes
→ component re-renders
→ useEffect runs
→ title updates
```

# 44. useState Hook

Tag: **[BOTH]**

This is the **most fundamental Hook in React**. Almost every React component uses it.

Before learning syntax, we must understand **why `useState` exists**.

---

# 1. Problem — Why State Is Needed

Web UI is **not static**.

Examples:

- Counter increasing
- Form inputs changing
- Login status changing
- API data loading
- Cart items updating

Example UI:

```
Count: 0
[ Increase ]
```

User clicks button:

```
Count: 1
```

The UI **changes dynamically**.

This changing data is called:

```
STATE
```

State = **data that changes over time and affects UI rendering**

---

# 2. React Mental Model

React UI is **a function of state**.

```
UI = f(state)
```

Example:

```
state = 0 → UI shows 0
state = 5 → UI shows 5
```

Whenever state changes:

```
React re-renders component
```

---

# 3. What useState Does

`useState` allows a component to **store and update state**.

Syntax:

```javascript
const [state, setState] = useState(initialValue);
```

Breakdown:

| Part         | Meaning                  |
| ------------ | ------------------------ |
| state        | current value            |
| setState     | function to update value |
| initialValue | starting value           |

Example:

```javascript
const [count, setCount] = useState(0);
```

Meaning:

```
count → current value
setCount → update function
0 → initial value
```

---

# 4. Basic Example

### Counter Example

```jsx
import { useState } from "react";

function Counter() {
  // create state variable with initial value 0
  const [count, setCount] = useState(0);

  return (
    <div>
      {/* display state */}
      <h1>{count}</h1>

      {/* update state */}
      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}

export default Counter;
```

---

# 5. Step-by-Step Execution

Initial render:

```
count = 0
```

UI:

```
0
[ Increase ]
```

User clicks button:

```
setCount(count + 1)
```

React:

```
count becomes 1
component re-renders
UI updates
```

Result:

```
1
```

---

# 6. Destructuring Pattern

`useState` returns **an array**.

Example:

```javascript
const stateArray = useState(0);
```

Actual value:

```
[0, function]
```

So we extract using **array destructuring**.

```javascript
const [count, setCount] = useState(0);
```

Meaning:

```
count = 0
setCount = update function
```

---

# 7. Multiple States

You can create **multiple state variables**.

Example:

```jsx
import { useState } from "react";

function Profile() {
  const [name, setName] = useState("Tanish");
  const [age, setAge] = useState(20);

  return (
    <div>
      <h1>{name}</h1>
      <h2>{age}</h2>

      <button onClick={() => setAge(age + 1)}>Increase Age</button>
    </div>
  );
}
```

State structure:

```
name = "Tanish"
age = 20
```

Each state is **independent**.

---

# 8. Updating State Correctly

Wrong approach:

```javascript
count = count + 1;
```

React will **NOT detect this change**.

Correct way:

```javascript
setCount(count + 1);
```

Because React tracks **setState functions**.

---

# 9. Functional State Updates

Sometimes state updates depend on **previous value**.

Example:

Bad:

```javascript
setCount(count + 1);
setCount(count + 1);
```

Result:

```
+1 only
```

Why?

React batches updates.

Correct:

```javascript
setCount((prev) => prev + 1);
setCount((prev) => prev + 1);
```

Now result:

```
+2
```

Example:

```jsx
<button
  onClick={() => {
    setCount((prev) => prev + 1);
    setCount((prev) => prev + 1);
  }}
>
  Increase Twice
</button>
```

---

# 10. State With Objects

State can store **objects**.

Example:

```jsx
const [user, setUser] = useState({
  name: "Tanish",
  age: 20,
});
```

Update:

```javascript
setUser({
  ...user,
  age: 21,
});
```

Important:

```
React state must be immutable
```

Never modify directly.

Wrong:

```javascript
user.age = 21;
```

Correct:

```
create new object
```

---

# 11. State With Arrays

Example:

```jsx
const [items, setItems] = useState([]);
```

Add item:

```javascript
setItems([...items, "Apple"]);
```

Explanation:

```
spread old array
add new element
create new array
```

---

```jsx
import { useState } from "react";

```

# 14. When NOT to Use State

Do not use state for:

```
constants
derived values

```

Example:

```javascript

```

```javascript
const sum = a + b;
```

# 45. useEffect Hook

Tag: **[BOTH]**

This hook is used for handling **side effects** in React components.

Before learning syntax, we must understand **why `useEffect` exists**.

---

# 1. Problem — Why useEffect Is Needed

React components are meant to **render UI**.

But real applications must also perform tasks like:

- Fetching API data
- Setting timers
- Updating document title
- Listening to events
- Subscribing to services

These operations are called:

```
Side Effects
```

### What is a Side Effect?

A **side effect** is anything that affects something **outside the component render process**.

Examples:

```
Fetching data from server
Changing browser title
Listening to window events
Saving data to localStorage
```

React rendering must stay **pure**, meaning:

```
UI output depends only on state and props
```

But side effects are necessary. So React provides **useEffect**.

---

# 2. What useEffect Does

`useEffect` allows a component to run code **after rendering**.

Syntax:

```javascript
useEffect(callbackFunction, dependencyArray);
```

Breakdown:

| Part             | Meaning                   |
| ---------------- | ------------------------- |
| callbackFunction | effect logic              |
| dependencyArray  | controls when effect runs |

Example:

```javascript
useEffect(() => {
  console.log("Component rendered");
});
```

This runs **after every render**.

---

# 3. Basic Example

### Update Page Title

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // update browser tab title
    document.title = `Count: ${count}`;
  });

  return (
    <div>
      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}

export default Counter;
```

Flow:

```
User clicks button
↓
count changes
↓
component re-renders
↓
useEffect runs
↓
title updates
```

---

# 4. React Lifecycle vs useEffect

Before hooks, React used **lifecycle methods**.

### Class Component Example

```jsx
class App extends React.Component {
  componentDidMount() {
    console.log("Component mounted");
  }

  componentDidUpdate() {
    console.log("Component updated");
  }

  componentWillUnmount() {
    console.log("Component removed");
  }
}
```

These methods controlled **component lifecycle**.

Hooks simplified this using **useEffect**.

---

# 5. How useEffect Replaces Lifecycle

`useEffect` can replace multiple lifecycle methods.

Example:

```jsx
useEffect(() => {
  console.log("component mounted or updated");
});
```

Equivalent to:

```
componentDidMount
componentDidUpdate
```

---

# 6. Example — Timer

```jsx
import { useEffect } from "react";

function Timer() {
  useEffect(() => {
    console.log("Timer started");
  });

  return <h1>Timer Component</h1>;
}
```

This runs **after every render**.

But this behavior is not always desired.

This is why React provides **dependency arrays**.

# 7. API Call Example

Very common real-world usage.

```jsx
import { useState, useEffect } from "react";

function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then((response) => response.json())
      .then((data) => setUsers(data));
  });

  return (
    <div>
      {users.map((user) => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}
```

Problem here:

```
API runs every render
```

Which can cause **infinite loops**.

Dependency arrays solve this.

# 8. Cleanup Function

Sometimes effects must be **cleaned up**.

Example:

- event listeners
- intervals
- subscriptions

Example:

```jsx
useEffect(() => {
  const interval = setInterval(() => {
    console.log("Running");
  }, 1000);

  return () => {
    clearInterval(interval);
  };
});
```

Explanation:

```
Effect starts interval
Cleanup stops interval
```

Cleanup runs when:

```
component unmounts
or effect re-runs
```

```
1 Render UI
2 Paint UI to screen
3 Run useEffect
```

Important:

```
useEffect runs AFTER render
```

This prevents UI blocking.

# 46. Dependency Array Logic

Tag: **[BOTH]**

This topic is **critical for mastering `useEffect`**.

Most bugs in React applications happen because developers **misunderstand the dependency array**.

So we must understand **exactly how React decides when an effect runs**.

---

# 1. Problem — When Should an Effect Run?

Consider this component:

```jsx
function Example() {
  useEffect(() => {
    console.log("Effect executed");
  });

  return <h1>Hello</h1>;
}
```

What happens?

Every render runs the effect.

Example render cycle:

```
Render
↓
Effect runs

Render again
↓
Effect runs again
```

But many times we want different behavior:

Examples:

```
Run effect ONLY once
Run effect when some value changes
Run effect when multiple values change
```

This control is handled by:

```
Dependency Array
```

---

# 2. What is the Dependency Array?

Syntax:

```javascript
useEffect(effectFunction, dependencies);
```

Example:

```javascript
useEffect(() => {
  console.log("Effect");
}, [count]);
```

Here:

```
[count] = dependency array
```

React watches values inside this array.

If any value changes → effect runs.

---

# 3. Three Dependency Patterns

There are **3 main patterns**.

```
1 No dependency array
2 Empty dependency array
3 Dependency values
```

Each behaves differently.

---

# 4. Case 1 — No Dependency Array

Example:

```jsx
useEffect(() => {
  console.log("Runs every render");
});
```

Behavior:

```
Component renders
↓
Effect runs
↓
Component renders again
↓
Effect runs again
```

Equivalent lifecycle:

```
componentDidMount
componentDidUpdate
```

Use case:

Rarely used.

Example:

```
logging
debugging
```

---

# 5. Case 2 — Empty Dependency Array

Example:

```jsx
useEffect(() => {
  console.log("Runs only once");
}, []);
```

Behavior:

```
Component mounts
↓
Effect runs
↓
Never runs again
```

Equivalent lifecycle:

```
componentDidMount
```

Common use cases:

```
API calls
initial setup
analytics tracking
```

Example:

```jsx
import { useEffect } from "react";

function App() {
  useEffect(() => {
    console.log("Component mounted");
  }, []);

  return <h1>Hello</h1>;
}
```

---

# 6. Case 3 — Dependency Values

Example:

```jsx
useEffect(() => {
  console.log("Count changed");
}, [count]);
```

Behavior:

```
Initial render → effect runs
count changes → effect runs again
other state changes → effect does NOT run
```

Example component:

```jsx
import { useState, useEffect } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("Count updated:", count);
  }, [count]);

  return (
    <div>
      <h1>{count}</h1>

      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}
```

Flow:

```
count = 0
↓
effect runs

click button
↓
count = 1
↓
effect runs again
```

---

# 7. Multiple Dependencies

Dependency array can contain multiple values.

Example:

```jsx
useEffect(() => {
  console.log("Name or Age changed");
}, [name, age]);
```

Effect runs when:

```
name changes
OR
age changes
```

Example component:

```jsx
import { useState, useEffect } from "react";

function Profile() {
  const [name, setName] = useState("Alex");
  const [age, setAge] = useState(20);

  useEffect(() => {
    console.log("Profile updated");
  }, [name, age]);

  return (
    <div>
      <button onClick={() => setName("John")}>Change Name</button>

      <button onClick={() => setAge(age + 1)}>Increase Age</button>
    </div>
  );
}
```

---

# 8. Dependency Comparison (How React Checks Changes)

React compares dependencies using:

```
Object.is()
```

Meaning:

```
Primitive values → compared by value
Objects / arrays → compared by reference
```

Example:

```javascript
const obj1 = { name: "Alex" };
const obj2 = { name: "Alex" };

obj1 === obj2;
```

Result:

```
false
```

Because they are different references.

This causes many React bugs.

---

# 9. Infinite Loop Example

Common mistake:

```jsx
useEffect(() => {
  setCount(count + 1);
}, [count]);
```

Flow:

```
count changes
↓
effect runs
↓
setCount runs
↓
count changes again
↓
effect runs again
```

Result:

```
infinite loop
```

Example:

```jsx

  console.log("Effect started");

  return () => {
    console.log("Cleanup running");
  };
}, [count]);
```

Flow:

```
count changes
↓
cleanup runs
↓
effect runs again
```

Cleanup runs:

```
before next effect
OR
component unmount
```

# 1. Problem — Why Data Fetching Is Special in React

React components render UI.

But API calls are **side effects**.

Example:

```

↓

↓
Update state
↓
Re-render UI
```

Important rule:

```

```

Bad example:

```jsx
function Users() {
  const data = fetch("https://api.com/users");

```

Problems:


# 3. Basic API Fetch Example

```jsx
import { useState, useEffect } from "react";

function Users() {
  // state to store API data
  const [users, setUsers] = useState([]);

  useEffect(() => {
    // fetch data from API
    fetch("https://jsonplaceholder.typicode.com/users")
      .then((response) => response.json())
      .then((data) => {
        // store data in state
        setUsers(data);
      });

  return (
    <div>
      <h1>User List</h1>

      {users.map((user) => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}

export default Users;
```

# 6. Using Async / Await (Cleaner Pattern)

Instead of `.then()` chains.

Better version:

```jsx
import { useState, useEffect } from "react";

function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    async function fetchUsers() {
      const response = await fetch(
        "https://jsonplaceholder.typicode.com/users",
      );

      const data = await response.json();

      setUsers(data);
    }

    fetchUsers();
  }, []);

  return (
    <div>
      {users.map((user) => (
        <p key={user.id}>{user.name}</p>

    </div>
  );
}
```

Why we create a function:

```
useEffect callback cannot be async
```

# 7. Handling Errors

Real APIs may fail.


Better approach:

```jsx
import { useState, useEffect } from "react";

function Users() {
  const [users, setUsers] = useState([]);
  const [error, setError] = useState(null);

  useEffect(() => {
    async function fetchUsers() {
      try {
        const response = await fetch(
          "https://jsonplaceholder.typicode.com/users",
        );

        const data = await response.json();

        setUsers(data);
      } catch (err) {
        setError("Failed to fetch users");
      }
    }

    fetchUsers();
  }, []);

  if (error) {
    return <p>{error}</p>;
  }

  return (
    <div>
      {users.map((user) => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}
```

# 11. Abort Controller (Advanced)

If a component unmounts before API finishes:

```jsx
useEffect(() => {
```

# 48. Managing Loading State

Tag: **[BOTH]**

# 1. Problem — What Happens Without Loading State

Consider this component:

```jsx
function Users() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("/api/users")
      .then((res) => res.json())
      .then((data) => setUsers(data));

      {users.map((user) => (
        <p key={user.id}>{user.name}</p>

    </div>
  );
}
```

What happens initially?

```
Component renders
users = []
```

Result UI:

```
(empty page)
```

User may think:

```

Is data loading?
```

This creates **bad user experience**.

# 2. Solution — Loading State

We create an additional state:

```

```


Meaning:

```

false → API finished
```

# 3. Basic Loading Example

```jsx
import { useState, useEffect } from "react";

function Users() {
  // store API data
  const [users, setUsers] = useState([]);

  // loading indicator
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetchUsers() {
      const response = await fetch(
        "https://jsonplaceholder.typicode.com/users",
      );

      const data = await response.json();

      // store API data
      setUsers(data);

      // API finished
      setLoading(false);
    }

    fetchUsers();
  }, []);

  // show loading UI
  if (loading) {
    return <p>Loading users...</p>;
  }

  return (
    <div>
      {users.map((user) => (
        <p key={user.id}>{user.name}</p>
      ))}
    </div>
  );
}
```
