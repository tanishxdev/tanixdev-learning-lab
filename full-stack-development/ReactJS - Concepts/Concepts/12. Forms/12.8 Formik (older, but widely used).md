# 12.8 FORMIK

Formik was the most popular React form library before React Hook Form became dominant.

It is still used in many:

* Enterprise applications
* Legacy React projects
* Codebases before 2022
* Teams that prefer controlled form patterns

So you must understand Formik to work in real jobs.

We will go step-by-step:

---

# 1. What is Formik?

Formik is a form library that:

* Uses **controlled** form patterns
* Manages form state internally
* Provides a `useFormik` hook
* Simplifies validation
* Works well with Yup
* Handles form-level and field-level errors

Formik increases developer productivity but has **performance limitations**.

---

# 2. Why Do Some Companies Still Use Formik?

Reasons:

1. Old projects were built with Formik
2. Clear, simple API
3. Easy to learn
4. Great integration with Yup
5. Used widely in enterprise software until recently

Many interviewers still ask Formik questions.

# 4. Basic Formik Example

Create file:

`Formik1_Basic.jsx`

```jsx
import { useFormik } from "formik";

export default function Formik1_Basic() {
  const formik = useFormik({
    initialValues: {
      name: "",
      email: ""
    },
    onSubmit: (values) => {
      console.log("Form submitted:", values);
    }
  });

  return (
    <form onSubmit={formik.handleSubmit}>
      <h3>Formik Basic Example</h3>

      <input
        name="name"
        value={formik.values.name}
        onChange={formik.handleChange}
      />

      <input
        name="email"
        value={formik.values.email}
        onChange={formik.handleChange}
      />

      <button type="submit">Submit</button>
    </form>
  );
}
```

This is Formik's core pattern:

* `initialValues`
* `values`
* `handleChange`
* `handleSubmit`

# 5. Validation with Yup (Most Common Pattern)

Yup is almost always used with Formik.

Create schema:

`validation/formikSignupSchema.js`

```jsx
import * as Yup from "yup";

export const formikSignupSchema = Yup.object({
  name: Yup.string().required("Name required"),
  email: Yup.string().email("Invalid email").required("Email required"),
  password: Yup.string().min(8, "Min 8 characters")
});
```

Now integrate with Formik:

`Formik2_WithYup.jsx`

```jsx
import { useFormik } from "formik";
import { formikSignupSchema } from "./validation/formikSignupSchema";

export default function Formik2_WithYup() {
  const formik = useFormik({
    initialValues: {
      name: "",
      email: "",
      password: ""
    },
    validationSchema: formikSignupSchema,
    onSubmit: (values) => {
      console.log("Valid Data:", values);
    }
  });

  return (
    <form onSubmit={formik.handleSubmit}>
      <h3>Formik with Yup Validation</h3>

      <input
        name="name"
        value={formik.values.name}
        onChange={formik.handleChange}
      />
      {formik.errors.name && formik.touched.name && (
        <p style={{ color: "red" }}>{formik.errors.name}</p>
      )}

      <input
        name="email"
        value={formik.values.email}
        onChange={formik.handleChange}
      />
      {formik.errors.email && formik.touched.email && (
        <p style={{ color: "red" }}>{formik.errors.email}</p>
      )}

      <input
        name="password"
        type="password"
        value={formik.values.password}
        onChange={formik.handleChange}
      />
      {formik.errors.password && formik.touched.password && (
        <p style={{ color: "red" }}>{formik.errors.password}</p>
      )}

      <button type="submit">Create Account</button>
    </form>
  );
}
```

Formik automatically handles:

* errors
* touched fields
* validation
* submit

# 6. Field-Level Errors and Touched State

Formik tracks:

* `touched` → whether the user interacted with the field
* `errors` → validation output

Example inside JSX:

```jsx
{formik.errors.email && formik.touched.email && (
  <p>{formik.errors.email}</p>
)}
```

This is the standard Formik error pattern.

# 7. Common Use Case: Login Form

`Formik3_Login.jsx`

```jsx
import { useFormik } from "formik";

export default function Formik3_Login() {
  const formik = useFormik({
    initialValues: {
      email: "",
      password: ""
    },
    onSubmit: (values) => {
      console.log("Login:", values);
    }
  });

  return (
    <form onSubmit={formik.handleSubmit}>
      <h3>Formik Login</h3>

      <input
        name="email"
        placeholder="Email"
        value={formik.values.email}
        onChange={formik.handleChange}
      />

      <input
        name="password"
        type="password"
        placeholder="Password"
        value={formik.values.password}
        onChange={formik.handleChange}
      />

      <button type="submit">Login</button>
    </form>
  );
}
```

# 8. Formik vs React Hook Form (Very Important)

| Feature        | Formik               | React Hook Form |
| -------------- | -------------------- | --------------- |
| Re-renders     | Many                 | Minimal         |
| Performance    | Slower for big forms | Very fast       |
| Under the hood | Controlled           | Uncontrolled    |
| Validation     | Yup-based            | Yup-based       |
| Code size      | Larger               | Smaller         |
| Industry trend | Older                | Modern standard |

### Summary:

* Formik is easier for controlled-form thinking.
* React Hook Form is superior in **performance** and **DX**.

Most new projects use React Hook Form.

---

# 9. When You Should Use Formik

Use Formik when:

1. You work on an old codebase already using it
2. You prefer controlled components
3. You build smaller forms
4. You want simple Yup integration
5. You want manually controlled re-renders

---

# 10. When You Should NOT Use Formik

Avoid Formik when:

1. You have large forms (20+ fields)
2. Performance matters
3. You need fast typing
4. You need minimal re-renders
5. You are building an interactive UI

This is why modern developers prefer React Hook Form.
