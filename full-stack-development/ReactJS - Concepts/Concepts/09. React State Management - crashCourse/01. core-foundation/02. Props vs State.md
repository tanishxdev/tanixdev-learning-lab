## 1. Concept (Straight to the point)

### STATE

* Owned by the component itself
* Can change
* Managed inside the component
* Used for data that is dynamic

### PROPS

* Passed from parent to child
* Read-only
* Cannot be changed by child
* Used for configuration / communication

---

## 2. Core Mental Model

Think like this:

```
STATE  â†’ INTERNAL MEMORY (private)
PROPS  â†’ EXTERNAL INPUT (given)
```

### Simple Definition

| Aspect      | State                    | Props                        |
| ----------- | ------------------------ | ---------------------------- |
| Ownership   | Component itself         | Parent component             |
| Mutability  | Can change               | Read-only                    |
| Purpose     | Manage internal behavior | Pass data between components |
| Who updates | setState / setter        | Parent only                  |

## 5. Why This Distinction Matters

If you confuse props and state:

* You create buggy UI
* You break data flow
* You lose component predictability

Correct React mental flow:

```
Data flows DOWN via props  
Events flow UP via functions
```

## 6. Practical Code Example

### PropsVsState.jsx

```javascript
import React, { useState } from "react";

/*
====================================================
PROPS VS STATE DEMONSTRATION
====================================================
Parent component sends a label using props.
Child component manages counter using state.
====================================================
*/

function CounterDisplay({ title }) {
  // STATE: internal, managed by component itself
  const [count, setCount] = useState(0);

  function increaseCount() {
    setCount(count + 1);
  }

  return (
    <div>

      {/* PROPS: received from parent */}
      <h2>{title}</h2>

      {/* STATE: internal changing data */}
      <p>Count: {count}</p>

      <button onClick={increaseCount}>
        Increase
      </button>

    </div>
  );
}

/*
Parent Component
This sends data to child using PROPS
*/
function PropsVsState() {
  return (
    <div>
      <CounterDisplay title="Counter using Props + State" />
    </div>
  );
}

export default PropsVsState;
```
