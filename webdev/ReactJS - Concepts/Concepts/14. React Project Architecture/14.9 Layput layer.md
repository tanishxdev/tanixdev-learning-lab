

# 14.9 LAYOUTS LAYER

Layouts are a key architectural concept used in every real-world React and Next.js application.

They help structure your UI consistently and avoid repeating common components like:

* Navbar
* Sidebar
* Footer
* Page container
* Layout grid
* Responsive wrappers

Letâ€™s go deep and clear.

---

# 1. What is a Layout?

A **layout** is a wrapper component that defines the common structure shared across multiple pages.

Example:

* Every page should have a Navbar
* Dashboard pages should have a Sidebar
* Public pages should not have Sidebar
* Admin pages should have a different layout

Instead of adding Navbar in every page, you define a layout.

---

# 2. Why Layouts Are Needed

Without layouts:

* Code repeats on every page
* UI becomes inconsistent
* Hard to update the design
* Hard to manage page-level wrappers

Layouts solve all this.

---

# 3. Types of Layouts in Real Apps

Most apps need at least **two** layouts:

### A) PublicLayout (unauthenticated pages)

Contains:

* Navbar
* Footer

Used for:

* Home
* Login
* Register
* About
* Contact

### B) DashboardLayout (authenticated pages)

Contains:

* Sidebar
* Navbar
* Content area

Used for:

* Dashboard
* Settings
* Orders
* Profile

Advanced apps may also have:

* AdminLayout
* MinimalLayout
* AuthLayout

---

# 4. Folder Structure for Layouts

```
src/layouts/
    PublicLayout.jsx
    DashboardLayout.jsx
    AuthLayout.jsx
```

---

# 5. Example: PublicLayout

File: `src/layouts/PublicLayout.jsx`

```jsx
import Navbar from "../components/Navbar";
import Footer from "../components/Footer";

export default function PublicLayout({ children }) {
  return (
    <div>
      <Navbar />

      <main style={{ padding: "20px" }}>
        {children}
      </main>

      <Footer />
    </div>
  );
}
```

This layout wraps all public pages.

---

# 6. Example: DashboardLayout

File: `src/layouts/DashboardLayout.jsx`

```jsx
import Sidebar from "../components/Sidebar";
import Navbar from "../components/Navbar";

export default function DashboardLayout({ children }) {
  return (
    <div style={{ display: "flex" }}>

      <Sidebar />

      <div style={{ flex: 1 }}>
        <Navbar />

        <main style={{ padding: "20px" }}>
          {children}
        </main>
      </div>

    </div>
  );
}
```

This layout gives:

* Sidebar
* Top navbar
* Main content area

Exactly like admin panels and dashboards.

# 7. Using Layouts with React Router

This is an important pattern.

In `AppRouter.jsx`:

```jsx
import { Routes, Route } from "react-router-dom";
import PublicLayout from "../layouts/PublicLayout";
import DashboardLayout from "../layouts/DashboardLayout";

import HomePage from "../pages/HomePage";
import LoginPage from "../pages/LoginPage";
import DashboardPage from "../pages/DashboardPage";
import ProfilePage from "../pages/ProfilePage";

export default function AppRouter() {
  return (
    <Routes>

      {/* Public Layout */}
      <Route element={<PublicLayout />}>
        <Route path="/" element={<HomePage />} />
        <Route path="/login" element={<LoginPage />} />
      </Route>

      {/* Dashboard Layout */}
      <Route element={<DashboardLayout />}>
        <Route path="/dashboard" element={<DashboardPage />} />
        <Route path="/profile" element={<ProfilePage />} />
      </Route>

    </Routes>
  );
}
```

React Router v6 allows nested layouts using the `element` wrapper.

---

# 8. Outlet Component (Important)

Layouts must include `<Outlet />` to render nested pages.

Modify PublicLayout:

```jsx
import { Outlet } from "react-router-dom";

export default function PublicLayout() {
  return (
    <div>
      <Navbar />
      <main><Outlet /></main>
      <Footer />
    </div>
  );
}
```

Same for DashboardLayout.

`Outlet` is where the page content appears.

---

# 9. Layouts Make UI Clean and Scalable

Instead of doing this:

HomePage.jsx:

```jsx
<Navbar />
<main>Home</main>
<Footer />
```

LoginPage.jsx:

```jsx
<Navbar />
<main>Login</main>
<Footer />
```

ProfilePage.jsx:

```jsx
<Navbar />
<Sidebar />
<main>Profile</main>
```

You move all repetitive UI to layouts.

Pages become:

```jsx
export default function HomePage() {
  return <h2>Home</h2>;
}
```

Clean and minimal.

---

# 10. When You Should Create a New Layout

Create a layout when:

* Many pages share the same structure
* The UI wrapper repeats
* It is a different user group (admin vs customer)
* You need distinct navigation pieces
* You need consistent padding, spacing, grid

Layouts reduce UI duplication across the app.
