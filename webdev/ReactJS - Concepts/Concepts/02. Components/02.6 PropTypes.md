## 1. Problem Without PropTypes

In React, components receive data through `props`.

Without any validation:

- You can pass wrong data types
- Components can break silently
- Bugs appear at runtime
- Debugging becomes hard
- No self-documentation of component API

Example problem:

```jsx
<UserCard name={123} age="twenty" role={true} />
```

No error at compile time → UI breaks logically.

---

## 2. Core Concept

`PropTypes` is a **runtime type-checking system** for React props.

It helps you:

- Validate props types
- Enforce component contracts
- Catch bugs early
- Document component usage
- Improve reliability

Definition:

> PropTypes define what type of data a component expects.

## 6. Code Implementation

### A. Component Without PropTypes (Unsafe)

```jsx
function UserCard({ name, age, role }) {
  return (
    <div>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
      <p>Role: {role}</p>
    </div>
  );
}
```

---

### B. Component With PropTypes (Safe)

```jsx
import PropTypes from "prop-types";

function UserCard({ name, age, role }) {
  return (
    <div>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
      <p>Role: {role}</p>
    </div>
  );
}

UserCard.propTypes = {
  name: PropTypes.string.isRequired,
  age: PropTypes.number.isRequired,
  role: PropTypes.string.isRequired,
};

export default UserCard;
```

---

## 7. How PropTypes Works

When component renders:

1. React reads `UserCard.propTypes`
2. It checks incoming props
3. If mismatch → warning in console
4. App does not crash
5. Developer gets error feedback

Important:

> PropTypes only warn in development mode.

---

## 8. Output Behavior

Correct usage:

```jsx
<UserCard name="Tanish" age={21} role="Frontend Dev" />
```

No warning.

Wrong usage:

```jsx
<UserCard name={21} age="twenty" role={true} />
```

Console warning:

```txt
Warning: Failed prop type: Invalid prop `name` of type `number` supplied to `UserCard`, expected `string`.
```

## 11. Advanced PropTypes

### Object shape

```js
user: PropTypes.shape({
  name: PropTypes.string,
  age: PropTypes.number,
});
```

### Array of types

```js
items: PropTypes.arrayOf(PropTypes.string);
```

### One of values

```js
role: PropTypes.oneOf(["admin", "user", "guest"]);
```

### One of types

```js
id: PropTypes.oneOfType([PropTypes.string, PropTypes.number]);
```

## 18. PropTypes vs TypeScript

| PropTypes        | TypeScript            |
| ---------------- | --------------------- |
| Runtime checking | Compile-time checking |
| Dev warnings     | Build errors          |
| JS based         | Type system           |
| Optional         | Structural typing     |
