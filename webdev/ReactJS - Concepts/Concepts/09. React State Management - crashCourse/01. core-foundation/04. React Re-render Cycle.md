## 1. Concept

A re-render happens when React decides a component must update its UI because something changed.

React re-renders when:

* State changes
* Props change
* Parent re-renders

Re-render = React runs the component function again.

---

## 2. Core Mental Model

React Component = Normal JavaScript Function

Every re-render:

```
Component() runs again
Calculates new JSX
Compares with old UI (Virtual DOM)
Updates only changed parts
```

So React is not refreshing the page.
It is re-executing the component logic.

---

## 3. Re-render Flow (Step-by-step)

```
Initial Render:
Component function runs
JSX created
UI painted

State change occurs:
setState called
React schedules re-render
Component function runs again
New JSX generated
React compares old vs new
Only differences updated on screen
```

---

## 4. What Triggers Re-render

| Trigger                | Causes Re-render |
| ---------------------- | ---------------- |
| setState               | ‚úÖ Yes            |
| New props              | ‚úÖ Yes            |

## 5. Practical Demonstration Code

### ReRenderCycle.jsx

```javascript
import React, { useState } from "react";

/*
====================================================
RE-RENDER CYCLE DEMO
Helps observe how many times component runs
====================================================
*/

function ReRenderCycle() {

  // This log shows when component function runs
  console.log("Component Rendered");

  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h2>Count: {count}</h2>

      <button onClick={handleClick}>
        Increase Count
      </button>
    </div>
  );
}

export default ReRenderCycle;
```

## 8. Why React Re-renders Entire Function

Because React's philosophy is:

> UI should always represent current state

So it rebuilds UI snapshot every time.

This makes UI predictable and reliable.

## 12. Key Takeaways

* Re-render = component function runs again
* Triggered by state or props change
* JSX recalculated each time
* Only actual DOM differences update
