## 1. Problem Before JSX

In pure JavaScript + React without JSX, UI looks like this:

```javascript
React.createElement("h1", null, "Hello World");
```

Problems:

* Hard to read
* Not developer-friendly
* Looks nothing like HTML
* Complex for large UI
* Poor productivity

Writing nested UI becomes ugly and confusing.

---

## 2. What is JSX?

JSX = JavaScript XML

Definition:

> JSX is a syntax extension that allows writing HTML-like code inside JavaScript.

Important:

* JSX is NOT HTML
* It is converted into `React.createElement()` by Babel

JSX makes UI:

* Readable
* Declarative
* Similar to HTML
* Easy to debug

---

## 3. Real-World Intuition

Without JSX:
You write UI like machine code.

With JSX:
You write UI like human language.

# JSX RULES

(Problem → Concept → Folder → Code → Explanation → Output)

---

## 1. Problem Without JSX Rules

JSX looks like HTML, but it is **not HTML**.
If you write normal HTML habits, React will break or behave incorrectly.

Common beginner errors:

* Multiple root elements
* Using `class` instead of `className`
* Forgetting closing tags
* Writing JS directly without `{}`

React enforces strict rules to keep UI predictable.

---

## 2. Core Concept: JSX Rules

JSX follows JavaScript rules, not HTML rules.

### Main Rules:

1. Only ONE parent element allowed
2. All tags must be closed
3. Use className instead of class
4. JavaScript inside `{}`
5. camelCase attributes
6. Expressions only, not statements
7. Fragment needed for grouping

## 4. Code Implementation

### A. Correct JSX Rules Example

**src/components/JsxRules.jsx**

```jsx
// JsxRules.jsx
// Demonstrates correct and incorrect JSX usage.

function JsxRules() {

  const username = "Tanish";

  return (
    <>
      {/* Rule 1: Single Parent Element - Using Fragment */}
      <div>

        {/* Rule 2: All tags must be closed */}
        <h2>JSX Rules Demo</h2>
        <p>This paragraph is properly closed.</p>

        {/* Rule 3: className instead of class */}
        <div className="card">
          Styled Box
        </div>

        {/* Rule 4: JavaScript inside {} */}
        <p>Welcome, {username}</p>

        {/* Rule 5: camelCase attributes */}
        <button onClick={() => alert("Clicked")}>
          Click Me
        </button>

      </div>
    </>
  );
}

export default JsxRules;
```

## 5. Common Wrong Examples (For Understanding)

### ❌ Multiple Roots

```jsx
return (
  <h1>Title</h1>
  <p>Paragraph</p>
);
```

Fix:

```jsx
return (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
);
```

---

### ❌ Using HTML class

```jsx
<div class="box">Hello</div>
```

✅ Correct

```jsx
<div className="box">Hello</div>
```

---

### ❌ JS outside {}

```jsx
<p>2 + 2</p>
```

✅ Correct

```jsx
<p>{2 + 2}</p>
```

## 7. Why JSX Rules Exist

| Rule          | Reason                      |
| ------------- | --------------------------- |
| Single Parent | Consistent Virtual DOM tree |
| Closed Tags   | Predictable parsing         |
| className     | Avoid JS keyword conflict   |
| {} syntax     | Explicit JS rendering       |
| camelCase     | JS standard compliance      |
