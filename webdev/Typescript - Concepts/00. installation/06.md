## Introduction

TypeScript (TS) is a superset of JavaScript (JS) that adds static typing for better code quality and error detection. Unlike JS, TS files (`.ts`) **cannot run directly** in Node.js or browsersâ€”they must be compiled to JS first.

**Why Setup is Needed:**
- TS requires a compiler (`tsc`) to convert `.ts` to `.js`.
- Node.js executes the resulting `.js`.
- For learning/practice, we want instant runs (no manual compile) to mimic JS workflows.

**Key Tools:**
- **Node.js & npm**: Base runtime and package manager.
- **TypeScript**: Core compiler.
- **ts-node**: Runs `.ts` directly (compiles in memory).
- **VS Code**: IDE with extensions like Code Runner for easy execution.
- **Nodemon**: Optional for auto-reload on file changes.

**Windows-Specific Notes:**
- Use Command Prompt or PowerShell as admin for global installs.
- Paths like `C:\Users\YourName\AppData\Roaming\npm` may be involved for globals.
- Ensure VS Code is added to PATH during installation.

This guide ensures setups work for:
- Simple `.ts` scripts.
- Node.js integration (e.g., console apps).
- Learning folders without projects.
- Full projects with dependencies.

## Method 1: Global Installation (Quick Testing)

Install TypeScript system-wide for use anywhere. Good for simple `.ts` files without projects.

### How It Works
The `-g` flag installs `tsc` (TypeScript compiler) globally, making it available in any terminal. When you run `tsc file.ts`, it reads your `.ts` code, checks types, and outputs plain `.js`. Node.js then executes the `.js` as normal JavaScript. No project files neededâ€”pure command-line flow.

### Steps:
1. Open Command Prompt as admin.
2. Run:
   ```
   npm install -g typescript
   ```
3. Verify:
   ```
   tsc -v
   ```

### Running a Simple `.ts` File:
1. Create `simple.ts`:
   ```ts
   console.log("Hello from global TS!");
   ```
2. Compile: `tsc simple.ts` (creates `simple.js`).
3. Run: `node simple.js`.

### Use Cases:
- Quick experiments.
- Single-file scripts.

### Problems & Solutions:
- **Version Conflicts**: Use local installs for multiple projects.
- **'tsc' not recognized**: Add global npm path to Windows PATH.

**When to Use**: First-time try. **Avoid** for structured learning.

## Method 2: Local Project Installation (Manual Compilation)

Project-specific setup. Recommended for real apps where you compile to `.js`.

### How It Works
TypeScript is installed locally (in `node_modules`), isolated per project. `npx` runs the local `tsc`. `tsconfig.json` configures compilation rules (e.g., target JS version). `npx tsc` compiles all files based on config, outputting `.js` to a folder (e.g., `dist`). Node.js runs the clean JS outputâ€”ideal for production builds.

### Steps:
1. Create folder and open terminal.
2. `npm init -y`.
3. `npm install --save-dev typescript`.
4. `npx tsc --init`.

### Running:
1. Create `index.ts` (example code).
2. `npx tsc`.
3. `node index.js` (or `dist/index.js` if `outDir` set).

### Add Scripts:
```json
"scripts": { "build": "tsc", "start": "node index.js" }
```

### Use Cases:
- Node.js apps with dependencies.
- Production builds.

### Problems & Solutions:
- **No output folder**: Add `"outDir": "./dist"` in `tsconfig.json`.
- **Strict errors**: Set `"strict": false` temporarily.

**When to Use**: Structured projects. **Not ideal** for fast learning.

## Method 3: Local Setup with ts-node (Instant Run - Recommended for Learning)

Run `.ts` directly without compiling to `.js`. Perfect for practice, DSA, or concepts.

### How It Works
`ts-node` hooks into Node.js, reading `.ts` files, compiling them **in memory** (no `.js` files created), type-checking on-the-fly, and executing immediately. It uses your `tsconfig.json` for rules. `npx` ensures it runs the local version. This mimics running JS directlyâ€”fast feedback loop for learning.

### Steps:
1. In folder: `npm init -y`.
2. `npm install --save-dev typescript ts-node @types/node`.
3. `npx tsc --init`.
   Edit `tsconfig.json` (recommended for 2025 Node learning):
   ```json
   {
     "compilerOptions": {
       "target": "ES2022",
       "module": "CommonJS",
       "strict": true,
       "esModuleInterop": true,
       "skipLibCheck": true,
       "forceConsistentCasingInFileNames": true
     }
   }
   ```

### Running:
1. Create `intro.ts` (example code).
2. `npx ts-node intro.ts`.

### Use Cases:
- Learning folders with subdirs.
- Simple Node.js scripts without clutter.

### Problems & Solutions:
- **'ts-node' not recognized**: Always use `npx`.
- **Module errors**: Keep `"module": "CommonJS"` for basic Node compatibility.

**When to Use**: All learning casesâ€”fast, clean. (Alternative in 2025: Try `tsx` for modern ESM.)

## VS Code Configuration (Code Runner for One-Click Execution)

Make `.ts` run like `.js` with the â–¶ button.

### How It Works
Code Runner overrides default runners per language. For `.ts`, it executes your custom command (`npx ts-node`). When you click â–¶, VS Code spawns a terminal, runs the command on the current file, and shows outputâ€”bypassing manual terminal input.

### Steps:
1. Install Code Runner.
2. Open Settings (JSON).
3. Add:
   ```json
   "code-runner.executorMap": {
     "typescript": "npx ts-node"
   },
   "code-runner.runInTerminal": true
   ```
4. Restart VS Code.
5. Click â–¶ on `.ts` file.

### Alternative:
```json
"code-runner.executorMap": {
  "typescript": "cd $dir && npx ts-node $fileName"
}
```

### Use Cases:
- Direct run in any file.

### Problems & Solutions:
- **Not working**: Verify `npx ts-node -v`.

**When to Use**: Always for convenience.

## Advanced Setup: Auto-Run with Nodemon

Auto-reload on save.

### How It Works
Nodemon watches files for changes, restarting the process (`ts-node yourfile.ts`) automatically. Combines instant run with live reloadâ€”great for testing changes quickly.

### Steps:
1. `npm install --save-dev nodemon`.
2. Add script:
   ```json
   "scripts": { "watch": "nodemon --exec ts-node intro.ts" }
   ```
3. `npm run watch`.

### Problems & Solutions:
- **Wrong file**: Use relative paths.

**When to Use**: Intermediate/backend practice.
